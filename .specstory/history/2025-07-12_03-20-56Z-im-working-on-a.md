## 2. Full Configuration with All Options

```tsx
import { GlassUIProvider } from 'liquidify';
import 'liquidify/css';

function App() {
  return (
    <GlassUIProvider
      theme="light" // or "dark"
      glassConfig={{
        intensity: 0.8,        // Glass effect intensity (0-1)
        blur: 10,             // Blur amount in pixels
        saturation: 1.5,      // Color saturation multiplier
        enableMagnetic: true, // Enable magnetic hover effects
        enableSpecular: true  // Enable specular highlights
      }}
      hapticConfig={{
        enableVibration: true, // Enable haptic feedback
        enableAudio: true,     // Enable audio feedback
        enableVisual: true     // Enable visual feedback
      }}
    >
      {/* Your app content */}
    </GlassUIProvider>
  );
}
```

## 3. What GlassUIProvider Includes

The `GlassUIProvider` automatically sets up these context providers:
- **ThemeProvider**: Handles light/dark theme switching
- **LiquidGlassProvider**: Manages glass effect configurations
- **HapticProvider**: Controls haptic feedback settings
- **ToastProvider**: Enables toast notifications

## 4. If You Need Additional Global Config

```tsx
import { GlassUIProvider, GlobalConfigProvider } from 'liquidify';
import 'liquidify/css';

function App() {
  return (
    <GlassUIProvider>
      <GlobalConfigProvider
        config={{
          defaultVariant: 'glass',  // default component variant
          defaultSize: 'md',        // default component size
          enableAnimations: true,   // enable/disable animations
          enableA11y: true,         // accessibility features
          colorScheme: 'auto',      // 'auto' | 'light' | 'dark'
          reducedMotion: false,     // respect prefers-reduced-motion
          highContrast: false       // high contrast mode
        }}
      >
        {/* Your app content */}
      </GlobalConfigProvider>
    </GlassUIProvider>
  );
}
```

## 5. Next.js Configuration

## 6. Important Notes

- Always import the CSS file: `import 'liquidify/css'`
- The `GlassUIProvider` must wrap any component that uses hooks like `useToast`, `useTheme`, etc.
- All configuration props are optional - sensible defaults are provided
- For SSR (Next.js, Remix, etc.), the provider handles hydration automatically
