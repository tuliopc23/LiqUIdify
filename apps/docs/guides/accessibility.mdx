# Accessibility Excellence

Liquidify is built with accessibility as a core principle, ensuring all users can interact with your applications regardless of their abilities or assistive technologies. Every component meets WCAG 2.1 AA standards while maintaining Apple's accessibility excellence.

## üåü Accessibility-First Architecture

### Foundation Principles

Liquidify follows Apple's accessibility guidelines and web standards:

- **Inclusive by Default**: Every component ships with proper ARIA attributes
- **Keyboard First**: Complete keyboard navigation without requiring a mouse
- **Screen Reader Optimized**: Semantic HTML with descriptive announcements
- **Motor Accessibility**: Generous touch targets and reduced motion support
- **Cognitive Accessibility**: Clear visual hierarchy and predictable interactions

### Core Provider Setup

```tsx
import { GlassUIProvider, GlassLiveRegion } from "@liquidify/react";

function App() {
  return (
    <GlassUIProvider
      accessibilityConfig={{
        announceRouteChanges: true,      // Screen reader navigation
        announceThemeChanges: true,      // Theme switch announcements
        highContrastMode: false,         // Enhanced contrast mode
        reducedMotionMode: 'auto',       // Respect user preferences
        focusRingWidth: 2,               // Focus indicator thickness
        focusRingColor: '#007AFF',       // Apple system blue
        minimumTouchTarget: 44           // iOS HIG touch target size
      }}
    >
      <GlassLiveRegion aria-live="polite" />
      <AppContent />
    </GlassUIProvider>
  );
}
```

## ‚å®Ô∏è Comprehensive Keyboard Support

### Navigation Patterns

Liquidify implements Apple HIG keyboard patterns across all components:

<AccordionGroup>
  <Accordion title="Focus Management" icon="target">
    **Smart Focus Trapping**: Modal dialogs and drawers automatically trap focus within their boundaries, with proper restoration when dismissed.
    
    ```tsx
    <GlassModal open={isOpen} onOpenChange={setIsOpen}>
      <GlassModal.Content>
        {/* Focus automatically moves to first interactive element */}
        <GlassButton>First Action</GlassButton>
        <GlassInput placeholder="Text input" />
        <GlassButton onClick={() => setIsOpen(false)}>
          Close {/* Focus returns to trigger on close */}
        </GlassButton>
      </GlassModal.Content>
    </GlassModal>
    ```
  </Accordion>

  <Accordion title="Roving Tab Index" icon="keyboard">
    **Efficient Navigation**: Complex components like menus and tabs use roving tab index for single-tab navigation with arrow keys.
    
    ```tsx
    <GlassTabs defaultTab="overview">
      <GlassTabs.List>
        {/* Tab + Arrow Keys for navigation */}
        <GlassTabs.Trigger value="overview">Overview</GlassTabs.Trigger>
        <GlassTabs.Trigger value="details">Details</GlassTabs.Trigger>
        <GlassTabs.Trigger value="settings">Settings</GlassTabs.Trigger>
      </GlassTabs.List>
    </GlassTabs>
    ```
  </Accordion>

  <Accordion title="Escape Key Handling" icon="escape">
    **Consistent Dismissal**: All overlay components respond to Escape key for intuitive dismissal patterns.
    
    - **Modals**: Close and restore focus to trigger
    - **Dropdowns**: Close and maintain trigger focus
    - **Tooltips**: Dismiss without focus change
    - **Command Palettes**: Close and clear search state
  </Accordion>
</AccordionGroup>

### Dynamic Content Announcements

```tsx
import { useAnnouncement } from "@liquidify/react";

function DataTable() {
  const { announce } = useAnnouncement();
  const [sortColumn, setSortColumn] = useState('name');
  
  const handleSort = (column) => {
    setSortColumn(column);
    announce(`Table sorted by ${column}`, 'polite');
  };
  
  return (
    <GlassTable
      data={tableData}
      onSort={handleSort}
      aria-label="User data table"
      aria-describedby="table-description"
    >
      <caption id="table-description">
        Table showing user information with sortable columns
      </caption>
      {/* Table content */}
    </GlassTable>
  );
}
```

### Loading States and Progress

```tsx
function AsyncOperation() {
  const [loading, setLoading] = useState(false);
  const [progress, setProgress] = useState(0);
  
  return (
    <div>
      <GlassButton 
        onClick={startOperation}
        disabled={loading}
        aria-describedby={loading ? "operation-status" : undefined}
      >
        {loading ? 'Processing...' : 'Start Operation'}
      </GlassButton>
      
      {loading && (
        <GlassProgress
          value={progress}
          aria-label="Operation progress"
          aria-describedby="operation-status"
        />
      )}
      
      <div id="operation-status" aria-live="polite" className="sr-only">
        {loading ? `Processing: ${progress}% complete` : ''}
      </div>
    </div>
  );
}
```

### Reduced Motion Support

Respecting user motion preferences while maintaining functionality:

```tsx
import { useReducedMotion } from "@liquidify/react";

function AnimatedComponent() {
  const reducedMotion = useReducedMotion();
  
  return (
    <GlassCard
      animate={!reducedMotion}
      transition={reducedMotion ? { duration: 0 } : { type: 'spring' }}
    >
      Content with respectful animations
    </GlassCard>
  );
}
```


### Automated Testing Integration

```typescript
// Jest + Testing Library accessibility tests
import { render, screen } from '@testing-library/react';
import { axe, toHaveNoViolations } from 'jest-axe';

expect.extend(toHaveNoViolations);

test('GlassButton has no accessibility violations', async () => {
  const { container } = render(
    <GlassButton>Click me</GlassButton>
  );
  
  const results = await axe(container);
  expect(results).toHaveNoViolations();
});

test('Component supports keyboard navigation', async () => {
  render(<GlassModal open={true}>Modal content</GlassModal>);
  
  // Test focus trapping
  await user.tab();
  expect(screen.getByRole('button', { name: /close/i })).toHaveFocus();
  
  // Test escape key
  await user.keyboard('{Escape}');
  expect(screen.queryByRole('dialog')).not.toBeInTheDocument();
});
```
