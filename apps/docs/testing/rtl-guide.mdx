title: React Testing Library
---

Usage patterns
- Prefer getByRole with name over getByText
- Use userEvent for realistic interactions (click, type, keyboard)
- Cleanup is automatic via setup.ts afterEach

Common imports
- import { render, screen, within } from '@testing-library/react'
- import userEvent from '@testing-library/user-event'

Custom render
- If providers are needed, use libs/components/src/test/test-utils.tsx

Examples
- Render and assert text
  - const { getByRole } = render(<Button>Save</Button>)
  - expect(getByRole('button', { name: /save/i })).toBeInTheDocument()

- Interactions
  - await userEvent.click(getByRole('button'))
  - await userEvent.type(getByLabelText(/email/i), 'user@example.com')

- Queries
  - getBy..., queryBy..., findBy... for sync/async needs

A11y matchers (jest-dom)
- toBeInTheDocument, toHaveAccessibleName, toBeDisabled, toHaveAttribute
