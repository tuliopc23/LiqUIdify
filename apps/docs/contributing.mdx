# Contributing to Liquidify

We welcome contributions to Liquidify! This guide will help you understand our development workflow, design principles, and quality standards to ensure your contributions align with our Apple HIG-inspired liquid glass design language.

## üöÄ Quick Start for Contributors

### Development Environment Setup

```bash
# Clone the repository
git clone https://github.com/tuliopc23/LiqUIdify.git
cd LiqUIdify

# Install dependencies
bun install

# Start development servers
bun run dev:all          # All development servers
bun run storybook        # Component playground
bun run docs:dev         # Documentation site
```

### Essential Development Commands

```bash
# Type checking
bun run type-check       # Verify TypeScript compliance

# Code quality
bun run lint            # ESLint and Prettier
bun run format:fix      # Auto-format code

# Testing
bun run test           # Run test suite
bun run test:coverage  # Coverage reports

# Building
bun run build:lib      # Build component library
bun run build:all      # Build all packages
```

## üé® Design Principles

### Liquid Glass Design Language

Our components follow a sophisticated **liquid glass design language** inspired by Apple's Human Interface Guidelines:

- **Layered Transparency**: Multiple glass layers with varying opacity levels
- **Content-Aware Adaptation**: Glass effects that respond to underlying content
- **Specular Highlights**: Subtle light reflections that enhance depth perception
- **Physics-Based Motion**: Spring animations and momentum-based interactions

### Apple HIG Integration

All components must adhere to these Apple design principles:

- **Clarity**: Clear visual hierarchy and purposeful design
- **Deference**: Interface doesn't compete with content
- **Depth**: Realistic layering and dimensional effects
- **Accessibility**: WCAG 2.1 AA compliance minimum

## üìã Contribution Checklist

### Before Submitting a PR

- [ ] **TypeScript Compliance**: `bun run type-check` passes
- [ ] **Code Quality**: `bun run lint` and `bun run format:fix` applied
- [ ] **Test Coverage**: All tests pass with `bun run test`
- [ ] **Storybook Stories**: Component examples in Storybook
- [ ] **Documentation**: MDX documentation in `apps/docs/components/`
- [ ] **Accessibility**: Keyboard navigation and screen reader support
- [ ] **Design Review**: Follows liquid glass design principles

### Component Development Standards

#### Required Implementation

1. **TypeScript**: Comprehensive type definitions
2. **Accessibility**: ARIA attributes, keyboard navigation
3. **Glass Effects**: Appropriate backdrop blur and transparency
4. **Theme Support**: Light/dark mode compatibility
5. **Responsive Design**: Mobile-first approach
6. **Performance**: Optimized rendering and animations

#### Code Quality Standards

```tsx
// Example component structure
interface GlassComponentProps {
  variant?: 'primary' | 'secondary' | 'ghost'
  size?: 'sm' | 'md' | 'lg'
  children: React.ReactNode
  className?: string
  // ... other props
}

export const GlassComponent = forwardRef<HTMLElement, GlassComponentProps>(
  ({ variant = 'primary', size = 'md', children, className, ...props }, ref) => {
    // Implementation with glass effects
    return (
      <div
        ref={ref}
        className={clsx(
          'glass-base-styles',
          `glass-variant-${variant}`,
          `glass-size-${size}`,
          className
        )}
        {...props}
      >
        {children}
      </div>
    )
  }
)
```
