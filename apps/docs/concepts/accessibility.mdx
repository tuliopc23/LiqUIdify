# Accessibility in LiqUIdify

LiqUIdify is built with accessibility at its core, following WCAG 2.1 AA guidelines to ensure a high-quality experience for all users.

## Overview

Our accessibility implementation focuses on four key principles:

1. **Perceivable**: Information must be presentable in ways all users can perceive
2. **Operable**: Interface components must be operable by all users
3. **Understandable**: Information and operation must be understandable
4. **Robust**: Content must be robust enough for reliable interpretation

## WCAG 2.1 AA Compliance

### 1. Perceivable

#### Text Alternatives

- All non-text content has text alternatives
- Icons include descriptive aria-labels
- Images have meaningful alt text
- SVGs include title and desc elements

```tsx
// Good Example
\u003cimg
  src="/avatar.jpg"
  alt="User profile picture showing a smiling person"
  role="img"
/\u003e

// For decorative images
\u003cimg src="/decoration.svg" alt="" role="presentation" /\u003e
```

#### Time-Based Media

- Video content includes captions
- Audio content has transcripts
- Media controls are keyboard accessible

```tsx
\u003cvideo
  controls
  aria-label="Tutorial video"
\u003e
  \u003ctrack
    kind="captions"
    src="/captions.vtt"
    label="English captions"
  /\u003e
\u003c/video\u003e
```

#### Adaptable Content

- Content can be presented in different ways
- Maintains meaning when linearized
- Proper heading structure

```tsx

\u003cmain\u003e
  \u003ch1\u003eMain Title\u003c/h1\u003e
  \u003csection aria-labelledby="section1-heading"\u003e

    \u003cp\u003eContent...\u003c/p\u003e
  \u003c/section\u003e
\u003c/main\u003e
```

#### Distinguishable Content

- Color is not the only means of conveying information
- Sufficient contrast ratios (4.5:1 for normal text)
- Text can be resized up to 200%
- Images of text are avoided

```tsx

\u003cdiv
  role="status"
  className="status-error"
  aria-label="Error status"
\u003e
  \u003cErrorIcon /\u003e
```

### 2. Operable

#### Keyboard Accessible

- All functionality available from keyboard
- No keyboard traps
- Custom keyboard shortcuts documented


#### Enough Time

- Adjustable time limits
- Pause, stop, hide moving content
- No auto-updating without user control

```tsx
  return (
    \u003cdiv\u003e
      \u003cbutton
```

#### Seizures and Physical Reactions

- No content flashing more than 3 times per second
- Motion animation can be disabled
- Reduced motion support


### 4. Robust

#### Compatible

- Valid HTML/ARIA usage
