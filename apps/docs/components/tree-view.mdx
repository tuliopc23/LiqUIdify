## Props

| Prop              | Type                                          | Default | Description                              |
| ----------------- | --------------------------------------------- | ------- | ---------------------------------------- |
| `nodes`           | `TreeNode[]`                                  | -       | Array of tree nodes                      |
| `onNodeSelect`    | `(node: TreeNode) => void`                    | -       | Callback when node is selected           |
| `onNodeExpand`    | `(node: TreeNode, expanded: boolean) => void` | -       | Callback when node is expanded/collapsed |
| `selectedNodeId`  | `string`                                      | -       | ID of currently selected node            |
| `expandedNodeIds` | `string[]`                                    | -       | Array of expanded node IDs (controlled)  |
| `showIcons`       | `boolean`                                     | `true`  | Whether to show node icons               |
| `indentSize`      | `number`                                      | `20`    | Indentation size in pixels               |
| `className`       | `string`                                      | -       | Additional CSS classes                   |

### TreeNode Interface

| Property          | Type         | Description                                  |
| ----------------- | ------------ | -------------------------------------------- |
| `id`              | `string`     | Unique node identifier                       |
| `label`           | `string`     | Display label                                |
| `icon`            | `ReactNode`  | Optional icon element                        |
| `children`        | `TreeNode[]` | Child nodes                                  |
| `selectable`      | `boolean`    | Whether node can be selected (default: true) |
| `defaultExpanded` | `boolean`    | Whether node is expanded by default          |
| `data`            | `unknown`    | Additional node data                         |

### Controlled Expansion

```tsx
const [expandedIds, setExpandedIds] = useState(['folder1'])

<GlassTreeView
  nodes={nodes}
  expandedNodeIds={expandedIds}
  onNodeExpand={(node, expanded) => {
    setExpandedIds(prev =>
      expanded
        ? [...prev, node.id]
        : prev.filter(id => id !== node.id)
    )
  }}
/>
```

## Accessibility

- **ARIA Tree**: Implements proper `role="tree"` and `role="treeitem"` attributes
- **Keyboard Navigation**:
  - Arrow keys for navigation
  - Enter/Space to select
  - Right arrow to expand, left arrow to collapse
  - Home/End to jump to first/last item
- **Screen Reader**: Announces tree structure, selection, and expansion state
- **Focus Management**: Proper focus indicators and tab order
- **ARIA Attributes**: Uses `aria-expanded`, `aria-selected`, and `aria-level`

## Apple HIG Guidelines

### Visual Design

- **Hierarchy**: Clear visual hierarchy with proper indentation
- **Depth**: Subtle glass effects that don't interfere with content
- **Icons**: Consistent icon usage for different node types
- **Typography**: Clear, readable text with appropriate contrast

### Interaction

- **Selection**: Clear visual feedback for selected items
- **Expansion**: Smooth animations for expand/collapse
- **Hover**: Subtle hover effects for better discoverability
- **Touch**: Appropriate touch targets for mobile devices

### Layout

- **Spacing**: Consistent spacing between items and levels
- **Alignment**: Proper alignment of icons, text, and expansion indicators
- **Responsive**: Adapts to different container sizes
- **Scrolling**: Handles overflow gracefully

### Performance

- **Virtualization**: Consider virtual scrolling for large trees
- **Lazy Loading**: Load child nodes on demand when possible
- **Memoization**: Optimize re-renders for large datasets
- **Animation**: Smooth but performant animations

### Best Practices

- Use consistent icons for similar node types
- Provide meaningful labels for all nodes
- Consider the depth of your tree structure
- Test keyboard navigation thoroughly
- Ensure proper contrast for all states
- Handle loading states for dynamic content
- Provide search/filter functionality for large trees
