# Badge

A glassmorphic badge component for displaying status, counts, labels, and other small pieces of information. Features semantic color variants and smooth glass effects.

## Props

| Prop        | Type        | Default   | Description                 |
| ----------- | ----------- | --------- | --------------------------- | -------- | ----------- | -------------------------------------------- |
| `variant`   | `"default"  | "success" | "warning"                   | "error"` | `"default"` | Semantic variant affecting color and styling |
| `children`  | `ReactNode` | -         | **Required.** Badge content |
| `className` | `string`    | -         | Additional CSS classes      |

## Usage Patterns

### Status Dashboard

```tsx
function StatusDashboard({ services }) {
  const getStatusVariant = (status) => {
    switch (status) {
      case "operational":
        return "success";
      case "degraded":
        return "warning";
      case "outage":
        return "error";
      default:
        return "default";
    }
  };

  return (
    <div className="space-y-4">
      <h3 className="font-semibold">Service Status</h3>
      {services.map((service) => (
        <div
          key={service.id}
          className="flex items-center justify-between p-3 bg-white/5 rounded-lg"
        >
          <span className="font-medium">{service.name}</span>
          <GlassBadge variant={getStatusVariant(service.status)}>
            {service.status}
          </GlassBadge>
        </div>
      ))}
    </div>
  );
}
```

## Accessibility

The GlassBadge component follows WCAG 2.1 guidelines:

### Visual Design

- High contrast colors for readability
- Sufficient color contrast ratios (4.5:1 minimum)
- Clear typography with appropriate sizing
- Color is not the sole indicator of meaning

### Content

- Concise, meaningful text
- Descriptive labels for status indicators
- Consistent terminology across the application
- Screen reader friendly content

### Semantic HTML

- Uses appropriate HTML elements
- Proper ARIA labels when needed
- Meaningful content structure

## Apple HIG Guidelines

Following Apple's Human Interface Guidelines for labels and indicators:

- **Clarity**: Clean, readable typography
- **Depth**: Subtle glass effects create visual hierarchy
- **Color**: Semantic colors that convey meaning

- **Concise**: Brief, scannable text
- **Meaningful**: Clear purpose and context
- **Consistent**: Uniform styling and terminology

### Interaction

- **Subtle**: Non-intrusive visual presence
- **Contextual**: Appears where relevant
- **Responsive**: Adapts to different screen sizes

## Best Practices

### Content Guidelines

- Keep text short and descriptive
- Use consistent terminology
- Avoid technical jargon
- Consider internationalization

### Visual Hierarchy

- Use semantic variants appropriately
- Don't overuse badges in a single view
- Maintain consistent sizing
- Group related badges logically

### Performance

- Avoid excessive badge animations
- Use appropriate color variants
- Consider badge density in lists
- Test with various content lengths

### Responsive Design

- Ensure readability on all screen sizes
- Consider badge wrapping behavior
- Maintain touch-friendly spacing
- Test with different text lengths

### Task Management

```tsx
function TaskItem({ task }) {
  const getPriorityVariant = (priority) => {
    switch (priority) {
      case "high":
        return "error";
      case "medium":
        return "warning";
      case "low":
        return "success";
      default:
        return "default";
    }
  };

  const getStatusVariant = (status) => {
    switch (status) {
      case "completed":
        return "success";
      case "in-progress":
        return "warning";
      case "blocked":
        return "error";
      default:
        return "default";
    }
  };

  return (
    <div className="flex items-center justify-between p-3 bg-white/5 rounded-lg">
      <div className="flex-1">
        <h4 className="font-medium mb-1">{task.title}</h4>
        <p className="text-sm text-[#007AFF]">{task.description}</p>
      </div>
      <div className="flex gap-2">
        <GlassBadge variant={getPriorityVariant(task.priority)}>
          {task.priority}
        </GlassBadge>
        <GlassBadge variant={getStatusVariant(task.status)}>
          {task.status}
        </GlassBadge>
      </div>
    </div>
  );
}
```

### Analytics Dashboard

```tsx
function MetricCard({ metric }) {
  const getTrendVariant = (trend) => {
    if (trend > 0) return "success";
    if (trend < 0) return "error";
    return "default";
  };

  const getTrendText = (trend) => {
    if (trend > 0) return `+${trend}%`;
    if (trend < 0) return `${trend}%`;
    return "No change";
  };

  return (
    <div className="p-4 bg-white/10 rounded-lg">
      <div className="flex items-center justify-between mb-2">
        <h3 className="text-sm font-medium text-[#007AFF]">{metric.label}</h3>
        <GlassBadge variant={getTrendVariant(metric.trend)}>
          {getTrendText(metric.trend)}
        </GlassBadge>
      </div>
      <p className="text-2xl font-bold">{metric.value}</p>
      <p className="text-xs text-[#007AFF] mt-1">vs last period</p>
    </div>
  );
}
```
