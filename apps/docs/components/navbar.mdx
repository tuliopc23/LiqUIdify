## Overview

The Navbar component provides a responsive navigation header with Apple-inspired design principles. It supports various layouts, scroll behaviors, and adaptive transparency effects.

## Props

### Navbar Props

| Prop           | Type                              | Default    | Description                                    |
| -------------- | --------------------------------- | ---------- | ---------------------------------------------- |
| `brand`        | `ReactNode`                       | -          | Content for the left side (logo/brand)         |
| `children`     | `ReactNode`                       | -          | Navigation items                               |
| `actions`      | `ReactNode`                       | -          | Content for the right side (actions/user menu) |
| `position`     | `'static' \| 'fixed' \| 'sticky'` | `'sticky'` | Navbar positioning                             |
| `transparent`  | `boolean`                         | `false`    | Whether navbar is transparent                  |
| `height`       | `'sm' \| 'md' \| 'lg'`            | `'md'`     | Height variant                                 |
| `shadow`       | `boolean`                         | `true`     | Show shadow on scroll                          |
| `blurOnScroll` | `boolean`                         | `true`     | Blur background on scroll                      |
| `hideOnScroll` | `boolean`                         | `false`    | Hide navbar when scrolling down                |
| `bgColor`      | `string`                          | -          | Background color (for non-transparent)         |
| `zIndex`       | `number`                          | `50`       | Z-index value                                  |

### NavbarItem Props

| Prop       | Type         | Default | Description                |
| ---------- | ------------ | ------- | -------------------------- |
| `href`     | `string`     | -       | Link destination           |
| `active`   | `boolean`    | `false` | Active state               |
| `onClick`  | `() => void` | -       | Click handler              |
| `children` | `ReactNode`  | -       | **Required.** Item content |
| `disabled` | `boolean`    | `false` | Disabled state             |

## Usage Patterns

### Basic Navigation

```tsx
<Navbar brand={<Logo />} actions={<UserMenu />}>
  <NavbarItem href="/home">Home</NavbarItem>
  <NavbarItem href="/about">About</NavbarItem>
  <NavbarItem href="/contact">Contact</NavbarItem>
</Navbar>
```

### Fixed Transparent Header

```tsx
<Navbar
  position="fixed"
  transparent
  blurOnScroll
  brand={<Brand />}
  actions={<Actions />}
>
  <NavbarItem>Features</NavbarItem>
  <NavbarItem>Pricing</NavbarItem>
  <NavbarItem>Support</NavbarItem>
</Navbar>
```

### Auto-Hide on Scroll

```tsx
<Navbar hideOnScroll brand={<Logo />} actions={<MobileMenu />}>
  <NavbarItem>Dashboard</NavbarItem>
  <NavbarItem>Analytics</NavbarItem>
  <NavbarItem>Settings</NavbarItem>
</Navbar>
```

### Compact Mobile Layout

```tsx
<Navbar
  height="sm"
  brand={<CompactLogo />}
  actions={
    <button className="md:hidden">
      <Menu className="h-5 w-5" />
    </button>
  }
>
  <div className="hidden md:flex space-x-1">
    <NavbarItem>Home</NavbarItem>
    <NavbarItem>Products</NavbarItem>
    <NavbarItem>About</NavbarItem>
  </div>
</Navbar>
```

## Accessibility

### WCAG Compliance

- **Landmark Navigation**: Uses semantic `<nav>` element
- **Keyboard Navigation**: Full keyboard support with Tab and Enter
- **Screen Readers**: Proper heading hierarchy and labels
- **Focus Management**: Visible focus indicators

### Implementation Details

- Navigation items use proper link semantics
- Active states are announced to screen readers
- Skip links can be added for keyboard users
- Color contrast meets WCAG AA standards

### Keyboard Support

| Key      | Action                  |
| -------- | ----------------------- |
| `Tab`    | Navigate between items  |
| `Enter`  | Activate focused item   |
| `Space`  | Activate focused button |
| `Escape` | Close any open menus    |

## Apple HIG Guidelines

### Visual Hierarchy

- **Clear Structure**: Brand, navigation, and actions are visually distinct
- **Consistent Spacing**: Uses 8pt grid system throughout
- **Typography Scale**: Appropriate font sizes for different screen sizes
- **Color Harmony**: Follows Apple's color guidelines

### Interaction Design

- **Smooth Transitions**: All state changes are animated
- **Hover States**: Subtle feedback for interactive elements
- **Active States**: Clear indication of current page/section
- **Loading States**: Skeleton loading for dynamic content

### Responsive Behavior

- **Adaptive Layout**: Automatically adjusts for different screen sizes
- **Touch Targets**: Minimum 44px touch targets on mobile
- **Safe Areas**: Respects device safe areas and notches
- **Orientation Support**: Works in both portrait and landscape

### Glass Effects

- **Backdrop Blur**: Creates depth and hierarchy
- **Transparency Layers**: Multiple levels of transparency
- **Specular Highlights**: Subtle light reflections
- **Color Tinting**: Adaptive color overlays

## Best Practices

### Performance

- Use `position: sticky` for better performance than fixed
- Implement scroll throttling for scroll-based effects
- Lazy load navigation items when possible
- Optimize images and icons in the navbar

### UX Guidelines

- Keep navigation items to 5-7 main categories
- Use clear, descriptive labels
- Maintain consistent navigation across pages
- Provide visual feedback for all interactions

### Mobile Considerations

- Prioritize most important actions in mobile view
- Use hamburger menu for secondary navigation
- Ensure touch targets are appropriately sized
- Consider thumb-friendly placement of actions

### SEO and Analytics

- Use proper heading hierarchy (h1, h2, etc.)
- Implement structured data for navigation
- Track navigation interactions for insights
- Ensure all links are crawlable
