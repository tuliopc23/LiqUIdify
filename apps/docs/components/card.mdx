# Card

A versatile glassmorphic card component with layered liquid glass effects and a compound component API. Perfect for displaying structured content with Apple-inspired depth and transparency.

## Live Examples

### Basic Usage

```tsx live

          <GlassCard.Content>
```

```tsx live
                <li key={index} className="flex items-center gap-2">
                  <span className="text-green-500">âœ“</span>

                </li>
              ))}
            </ul>

  );
}
```

## Props

### GlassCard

| Prop          | Type                                                     | Default     | Description                              |
| ------------- | -------------------------------------------------------- | ----------- | ---------------------------------------- |
| `size`        | `"sm" \| "md" \| "lg" \| "xl"`                           | `"md"`      | Card size affecting padding and spacing  |
| `variant`     | `"default" \| "elevated" \| "outlined" \| "interactive"` | `"default"` | Visual style variant                     |
| `interactive` | `boolean`                                                | `false`     | Enable interactive states (hover, focus) |
| `selected`    | `boolean`                                                | `false`     | Selected state styling                   |
| `children`    | `ReactNode`                                              | -           | **Required.** Card content               |
| `className`   | `string`                                                 | -           | Additional CSS classes                   |
| `onClick`     | `(event) => void`                                        | -           | Click event handler                      |

### Card Sub-components

All sub-components (`Header`, `Title`, `Description`, `Content`, `Footer`, `Actions`) accept:

| Prop        | Type        | Default | Description                            |
| ----------- | ----------- | ------- | -------------------------------------- |
| `children`  | `ReactNode` | -       | Component content                      |
| `className` | `string`    | -       | Additional CSS classes                 |
| `as`        | `string`    | -       | HTML element to render as (Title only) |

## Usage Patterns

### Product Cards

```tsx
import { GlassCard, GlassButton, GlassBadge } from "liquidify";
import { Star } from "lucide-react";

function ProductCard({ product }) {
  return (
    <GlassCard>
      <GlassCard.Header>
        <div className="aspect-video bg-gray-200 rounded-lg mb-3">
          <img
            src={product.image}
            alt={product.name}
            className="w-full h-full object-cover rounded-lg"
          />
        </div>
        <div className="flex items-start justify-between">
          <div className="flex-1">
            <GlassCard.Title>{product.name}</GlassCard.Title>
            <div className="flex items-center gap-1 mt-1">
              <Star className="w-4 h-4 fill-yellow-400 text-yellow-400" />
              <span className="text-sm">{product.rating}</span>
              <span className="text-sm text-gray-500">({product.reviews})</span>
            </div>
          </div>
          {product.isNew && <GlassBadge variant="success">New</GlassBadge>}
        </div>
      </GlassCard.Header>
      <GlassCard.Content>
        <p className="text-sm text-gray-600 mb-3">{product.description}</p>
        <div className="flex items-center justify-between">
          <span className="text-lg font-bold">${product.price}</span>
          {product.originalPrice && (
            <span className="text-sm text-gray-500 line-through">
              ${product.originalPrice}
            </span>
          )}
        </div>
      </GlassCard.Content>
      <GlassCard.Actions>
        <GlassButton variant="primary" fullWidth>
          Add to Cart
        </GlassButton>
      </GlassCard.Actions>
    </GlassCard>
  );
}
```

### Dashboard Widgets

```tsx
import { GlassCard, GlassBadge } from "liquidify";
import { TrendingUp, TrendingDown } from "lucide-react";

function MetricCard({ metric }) {
  const isPositive = metric.change > 0;

  return (
    <GlassCard size="sm">
      <GlassCard.Header>
        <div className="flex items-center justify-between">
          <GlassCard.Title className="text-sm">{metric.label}</GlassCard.Title>
          <GlassBadge variant={isPositive ? "success" : "error"}>
            {isPositive ? "+" : ""}
            {metric.change}%
          </GlassBadge>
        </div>
      </GlassCard.Header>
      <GlassCard.Content>
        <div className="flex items-center gap-2">
          <span className="text-2xl font-bold">{metric.value}</span>
          {isPositive ? (
            <TrendingUp className="w-5 h-5 text-green-500" />
          ) : (
            <TrendingDown className="w-5 h-5 text-red-500" />
          )}
        </div>
        <p className="text-xs text-gray-500 mt-1">vs last period</p>
      </GlassCard.Content>
    </GlassCard>
  );
}
```

## Accessibility

The GlassCard component follows WCAG 2.1 guidelines:

### Semantic Structure

- Proper heading hierarchy with Title components
- Logical content organization
- Clear visual hierarchy
- Meaningful content grouping

### Interactive Cards

- Keyboard navigation support
- Focus indicators for clickable cards
- Proper ARIA attributes
- Screen reader friendly structure

### Visual Accessibility

- High contrast ratios
- Clear visual boundaries
- Sufficient spacing between elements
- Readable typography

## Apple HIG Guidelines

Following Apple's Human Interface Guidelines for content containers:

### Visual Design

- **Clarity**: Clean content organization and typography
- **Depth**: Layered glass effects create visual hierarchy
- **Consistency**: Uniform styling across card variants

### Content

- **Scannable**: Easy to scan and understand
- **Organized**: Logical content structure
- **Focused**: Single purpose per card

### Interaction

- **Responsive**: Immediate feedback for interactive cards
- **Predictable**: Consistent behavior patterns
- **Contextual**: Appropriate for the content type

## Best Practices

### Content Organization

- Use clear, descriptive titles
- Keep descriptions concise
- Group related information
- Maintain consistent content structure

### Visual Hierarchy

- Use appropriate card sizes
- Limit content density
- Maintain consistent spacing
- Consider card relationships

### Interactive Design

- Make clickable cards obvious
- Provide clear hover states
- Use appropriate cursor styles
- Consider touch targets on mobile

### Performance

- Avoid heavy content in cards
- Consider lazy loading for images
- Optimize for large lists
- Use appropriate card variants
