## Overview

The Sidebar component provides a flexible navigation solution with support for collapsible sections, nested items, and various layout modes. It features Apple-inspired glass effects and smooth animations.

## Props

### Sidebar Props

| Prop                  | Type                             | Default  | Description                       |
| --------------------- | -------------------------------- | -------- | --------------------------------- |
| `children`            | `ReactNode`                      | -        | **Required.** Sidebar content     |
| `open`                | `boolean`                        | -        | Controlled open state             |
| `defaultOpen`         | `boolean`                        | `false`  | Default open state (uncontrolled) |
| `onOpenChange`        | `(open: boolean) => void`        | -        | Callback when open state changes  |
| `position`            | `'left' \| 'right'`              | `'left'` | Sidebar position                  |
| `width`               | `'sm' \| 'md' \| 'lg' \| 'full'` | `'md'`   | Sidebar width                     |
| `overlay`             | `boolean`                        | `true`   | Overlay behavior                  |
| `push`                | `boolean`                        | `false`  | Push content instead of overlay   |
| `collapsible`         | `boolean`                        | `false`  | Collapsible on desktop            |
| `collapsedWidth`      | `string`                         | `'4rem'` | Collapsed width when collapsible  |
| `header`              | `ReactNode`                      | -        | Header content                    |
| `footer`              | `ReactNode`                      | -        | Footer content                    |
| `zIndex`              | `number`                         | `40`     | Z-index value                     |
| `closeOnOverlayClick` | `boolean`                        | `true`   | Close on overlay click            |
| `closeOnEsc`          | `boolean`                        | `true`   | Close on escape key               |

### SidebarItem Props

| Prop        | Type         | Default | Description                      |
| ----------- | ------------ | ------- | -------------------------------- |
| `icon`      | `ReactNode`  | -       | Item icon                        |
| `label`     | `string`     | -       | **Required.** Item label         |
| `href`      | `string`     | -       | Link destination                 |
| `active`    | `boolean`    | `false` | Active state                     |
| `onClick`   | `() => void` | -       | Click handler                    |
| `children`  | `ReactNode`  | -       | Nested items                     |
| `collapsed` | `boolean`    | `true`  | Collapsed state for nested items |
| `disabled`  | `boolean`    | `false` | Disabled state                   |
| `badge`     | `ReactNode`  | -       | Badge content                    |

### SidebarSection Props

| Prop       | Type        | Default | Description                   |
| ---------- | ----------- | ------- | ----------------------------- |
| `title`    | `string`    | -       | Section title                 |
| `children` | `ReactNode` | -       | **Required.** Section content |

## Usage Patterns

### Basic Sidebar

```tsx
<Sidebar>
  <SidebarItem icon={<Home />} label="Home" />
  <SidebarItem icon={<Settings />} label="Settings" />
  <SidebarItem icon={<User />} label="Profile" />
</Sidebar>
```

### Controlled Sidebar

```tsx
const [isOpen, setIsOpen] = useState(false)

<Sidebar open={isOpen} onOpenChange={setIsOpen}>
  <SidebarItem icon={<Dashboard />} label="Dashboard" />
  <SidebarItem icon={<Analytics />} label="Analytics" />
</Sidebar>
```

### Right-Side Overlay

```tsx
<Sidebar position="right" overlay>
  <SidebarSection title="Navigation">
    <SidebarItem icon={<Home />} label="Home" />
    <SidebarItem icon={<About />} label="About" />
  </SidebarSection>
</Sidebar>
```

### Nested Navigation

```tsx
<Sidebar>
  <SidebarItem icon={<Folder />} label="Products">
    <SidebarItem label="Electronics" href="/electronics" />
    <SidebarItem label="Clothing" href="/clothing" />
    <SidebarItem label="Books" href="/books" />
  </SidebarItem>
</Sidebar>
```

### Push Layout Mode

```tsx
<Sidebar push collapsible>
  <SidebarItem icon={<Dashboard />} label="Dashboard" />
  <SidebarItem icon={<Settings />} label="Settings" />
</Sidebar>
```

## Accessibility

### WCAG Compliance

- **Landmark Navigation**: Uses semantic navigation landmarks
- **Keyboard Navigation**: Full keyboard support with arrow keys
- **Screen Readers**: Proper ARIA labels and tree structure
- **Focus Management**: Maintains focus within sidebar when open

### Implementation Details

- Uses `role="navigation"` for the sidebar container
- Nested items use proper tree navigation patterns
- Active states are announced to screen readers
- Escape key closes overlay sidebars
- Focus trap prevents focus from leaving modal sidebars

### Keyboard Support

| Key               | Action                 |
| ----------------- | ---------------------- |
| `Tab`             | Navigate between items |
| `Enter` / `Space` | Activate focused item  |
| `Arrow Up/Down`   | Navigate between items |
| `Arrow Right`     | Expand collapsed item  |
| `Arrow Left`      | Collapse expanded item |
| `Escape`          | Close overlay sidebar  |

## Best Practices

### Performance

- Use `React.memo` for sidebar items that don't change frequently
- Implement virtualization for very long navigation lists
- Lazy load nested content when possible
- Optimize animations for 60fps performance

### UX Guidelines

- Keep navigation hierarchy shallow (max 2-3 levels)
- Group related items in sections
- Use clear, descriptive labels
- Provide visual feedback for all interactions
- Consider the user's mental model when organizing items

### Mobile Considerations

- Use overlay mode on mobile devices
- Ensure touch targets are at least 44px
- Support swipe gestures for opening/closing
- Consider thumb-friendly placement of important items
- Provide clear visual indication of active states

### State Management

- Use controlled components for complex state
- Persist sidebar state in localStorage when appropriate
- Handle loading states gracefully
- Provide fallback content for empty states
