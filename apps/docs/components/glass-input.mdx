# Glass Input

The `GlassInput` component brings elegant glassmorphism styling to form inputs while maintaining full accessibility and functionality.

## Import

<CodeGroup>

```tsx Component Import
import { GlassInput } from '@liquidify/components'
```

```tsx Bundle Import  
import { GlassInput } from '@liquidify/components/forms'
```

</CodeGroup>

## Basic Usage

<CodeGroup>

```tsx Simple Input
<GlassInput 
  placeholder="Enter your email"
  type="email"
/>
```

```tsx With Label
<GlassFormField label="Email Address">
  <GlassInput 
    type="email"
    placeholder="john@example.com"
  />
</GlassFormField>
```

</CodeGroup>

## Props

<ResponseField name="variant" type="string" default="default">
  The visual style of the input
  - `"default"` - Standard input styling
  - `"search"` - Optimized for search interfaces
  - `"password"` - Enhanced password input
  - `"email"` - Email-specific styling
</ResponseField>

<ResponseField name="size" type="string" default="md">
  The size of the input
  - `"sm"` - Small (32px height)
  - `"md"` - Medium (40px height)
  - `"lg"` - Large (48px height)
</ResponseField>

<ResponseField name="leftIcon" type="ReactNode">
  Icon to display on the left side
</ResponseField>

<ResponseField name="rightIcon" type="ReactNode">
  Icon to display on the right side
</ResponseField>

<ResponseField name="clearable" type="boolean" default="false">
  Shows a clear button when input has value
</ResponseField>

<ResponseField name="loading" type="boolean" default="false">
  Shows loading spinner in the input
</ResponseField>

<ResponseField name="error" type="boolean" default="false">
  Applies error styling to the input
</ResponseField>

<ResponseField name="warning" type="boolean" default="false">
  Applies warning styling to the input
</ResponseField>

<ResponseField name="success" type="boolean" default="false">
  Applies success styling to the input
</ResponseField>

<ResponseField name="fullWidth" type="boolean" default="false">
  Makes input take full width of container
</ResponseField>

<ResponseField name="autoComplete" type="string">
  HTML autocomplete attribute value
</ResponseField>

<ResponseField name="maxLength" type="number">
  Maximum number of characters allowed
</ResponseField>

<ResponseField name="pattern" type="string">
  Regular expression pattern for validation
</ResponseField>

<ResponseField name="inputMode" type="string">
  Virtual keyboard type for mobile devices
</ResponseField>

<ResponseField name="glassEffect" type="object">
  Custom glass effect configuration
  ```tsx
  {
    intensity?: 'light' | 'medium' | 'strong' | 'ultra'
    blur?: boolean
    backdrop?: boolean
  }
  ```
</ResponseField>

## Accessibility

LiqUIdify inputs follow accessibility best practices:

- **Label Association**: Proper labeling through `htmlFor` or `aria-labelledby`
- **Error Announcements**: Screen readers announce validation states
- **Keyboard Navigation**: Full keyboard support with proper tab order
- **ARIA Attributes**: Comprehensive ARIA support for all states
- **Focus Management**: Clear focus indicators and focus trapping
- **Input Description**: Support for `aria-describedby`

### Accessibility Example

```tsx
<GlassFormField
  label="Email Address"
  helperText="We'll never share your email"
  error={emailError}
  required
>
  <GlassInput
    type="email"
    value={email}
    onChange={handleEmailChange}
    leftIcon={<Mail className="h-4 w-4" />}
    error={!!emailError}
    aria-invalid={!!emailError}
    aria-describedby="email-help email-error"
    required
  />
</GlassFormField>

{emailError && (
  <div id="email-error" role="alert" className="text-red-600 text-sm mt-1">
    {emailError}
  </div>
)}

<div id="email-help" className="text-gray-600 text-sm mt-1">
  We'll never share your email with anyone else.
</div>
```

## Theming

Customize input appearance using CSS variables:

```css
.custom-glass-input {
  --glass-input-bg: rgba(255, 255, 255, 0.1);
  --glass-input-border: rgba(255, 255, 255, 0.2);
  --glass-input-text: rgb(255, 255, 255);
  --glass-input-placeholder: rgba(255, 255, 255, 0.6);
  
  /* Focus states */
  --glass-input-focus-border: rgba(59, 130, 246, 0.5);
  --glass-input-focus-ring: rgba(59, 130, 246, 0.2);
  
  /* Error states */
  --glass-input-error-border: rgba(239, 68, 68, 0.5);
  --glass-input-error-bg: rgba(239, 68, 68, 0.1);
  
  /* Success states */
  --glass-input-success-border: rgba(34, 197, 94, 0.5);
  --glass-input-success-bg: rgba(34, 197, 94, 0.1);
}
```

## Performance Notes

- Input components are optimized with `React.memo`
- Debounced search functionality prevents excessive API calls
- Icon components should be properly sized (16px recommended)  
- Glass effects use GPU acceleration for smooth interactions
- Large forms benefit from field-level validation to avoid re-renders
