# Skeleton

The `GlassSkeleton` component provides elegant loading placeholders with glassmorphism effects. It includes multiple animation variants, shapes, and pre-built patterns for common use cases.

## Live Examples

### Basic Skeleton

import { useState, useEffect } from "react";

export default function Demo() {

        style={{

          border: "1px solid rgba(255,255,255,0.2)",

        }}
      >
        {loading ? (
          <div>
            <GlassSkeleton
              width="80%"
              height={24}
              variant="shimmer"
              style={{ marginBottom: 12 }}
            />
            <GlassSkeleton
              width="100%"
              height={16}
              variant="shimmer"
              style={{ marginBottom: 8 }}
            />
            <GlassSkeleton
              width="100%"
              height={16}
              variant="shimmer"
              style={{ marginBottom: 8 }}
            />
            <GlassSkeleton
              width="60%"
              height={16}
              variant="shimmer"
              style={{ marginBottom: 16 }}
            />

                display: "flex",

              style={{

                ‚≠ê {data.rating} ({data.reviews} reviews)
              </div>
            </div>
          </div>

    </div>

### Interactive Demo

```tsx live
import { useState } from "react";
import { GlassSkeleton, GlassButton } from "liquidify";

export default function Demo() {
  const [variant, setVariant] = useState("shimmer");
  const [shape, setShape] = useState("rectangle");
  const [animated, setAnimated] = useState(true);
  const [count, setCount] = useState(1);

  const variants = ["default", "shimmer", "pulse"];
  const shapes = ["rectangle", "circle", "line"];

  return (
    <div style={{ padding: 20 }}>
      <div
        style={{
          display: "flex",
          flexDirection: "column",
          gap: 16,
          marginBottom: 24,
        }}
      >
        <div>
          <h4 style={{ marginBottom: 8 }}>Variant</h4>
          <div style={{ display: "flex", gap: 8, flexWrap: "wrap" }}>
            {variants.map((v) => (
              <GlassButton
                key={v}
                size="sm"
                variant={variant === v ? "primary" : "ghost"}
                onClick={() => setVariant(v)}
              >
                {v}
              </GlassButton>
            ))}
          </div>
        </div>

        <div>
          <h4 style={{ marginBottom: 8 }}>Shape</h4>
          <div style={{ display: "flex", gap: 8, flexWrap: "wrap" }}>
            {shapes.map((s) => (
              <GlassButton
                key={s}
                size="sm"
                variant={shape === s ? "primary" : "ghost"}
                onClick={() => setShape(s)}
              >
                {s}
              </GlassButton>
            ))}
          </div>
        </div>

        <div>
          <h4 style={{ marginBottom: 8 }}>Options</h4>
          <div style={{ display: "flex", gap: 8, flexWrap: "wrap" }}>
            <GlassButton
              size="sm"
              variant={animated ? "primary" : "ghost"}
              onClick={() => setAnimated(!animated)}
            >
              Animated
            </GlassButton>
          </div>
        </div>

        <div>
          <h4 style={{ marginBottom: 8 }}>Count: {count}</h4>
          <div style={{ display: "flex", gap: 8 }}>
            <GlassButton
              size="sm"
              onClick={() => setCount(Math.max(1, count - 1))}
            >
              -
            </GlassButton>
            <GlassButton
              size="sm"
              onClick={() => setCount(Math.min(10, count + 1))}
            >
              +
            </GlassButton>
          </div>
        </div>
      </div>

      <div
        style={{
          padding: 20,
          border: "1px solid rgba(255,255,255,0.2)",
          borderRadius: 8,
        }}
      >
        <GlassSkeleton
          variant={variant}
          shape={shape}
          animated={animated}
          count={count}
          width={shape === "circle" ? 60 : 200}
          height={shape === "line" ? undefined : 60}
          spacing={12}
        />
      </div>
    </div>
  );
}
```

## Props

### GlassSkeleton

| Prop        | Type                                | Default       | Description                |
| ----------- | ----------------------------------- | ------------- | -------------------------- |
| `variant`   | `"default" \| "shimmer" \| "pulse"` | `"default"`   | Animation style            |
| `size`      | `"sm" \| "md" \| "lg" \| "xl"`      | `"md"`        | Predefined size            |
| `shape`     | `"rectangle" \| "circle" \| "line"` | `"rectangle"` | Shape of the skeleton      |
| `width`     | `string \| number`                  | `undefined`   | Custom width               |
| `height`    | `string \| number`                  | `undefined`   | Custom height              |
| `count`     | `number`                            | `1`           | Number of skeleton items   |
| `spacing`   | `number`                            | `8`           | Spacing between items (px) |
| `animated`  | `boolean`                           | `true`        | Enable animations          |
| `className` | `string`                            | `undefined`   | Additional CSS classes     |

### SkeletonText

| Prop            | Type     | Default     | Description            |
| --------------- | -------- | ----------- | ---------------------- |
| `lines`         | `number` | `3`         | Number of text lines   |
| `lastLineWidth` | `string` | `"60%"`     | Width of the last line |
| `className`     | `string` | `undefined` | Additional CSS classes |

### SkeletonCard

| Prop         | Type      | Default     | Description            |
| ------------ | --------- | ----------- | ---------------------- |
| `showAvatar` | `boolean` | `true`      | Show avatar skeleton   |
| `className`  | `string`  | `undefined` | Additional CSS classes |

### SkeletonTable

| Prop        | Type     | Default     | Description             |
| ----------- | -------- | ----------- | ----------------------- |
| `rows`      | `number` | `5`         | Number of table rows    |
| `columns`   | `number` | `4`         | Number of table columns |
| `className` | `string` | `undefined` | Additional CSS classes  |

## Usage Patterns

### Page Loading

```tsx
const PageSkeleton = () => (
  <div className="space-y-6">
    {/* Header */}
    <div className="flex items-center gap-4">
      <GlassSkeleton shape="circle" width={60} height={60} />
      <div className="flex-1">
        <GlassSkeleton width="40%" height={24} className="mb-2" />
        <GlassSkeleton width="60%" height={16} />
      </div>
    </div>

    {/* Content */}
    <SkeletonText lines={6} />

    {/* Cards */}
    <div className="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3 gap-4">
      {Array.from({ length: 6 }, (_, i) => (
        <SkeletonCard key={i} />
      ))}
    </div>
  </div>
);
```

## Accessibility

### WCAG Compliance

- **WCAG 2.1 AA**: Respects motion preferences and provides alternative content
- **Screen Readers**: Proper ARIA attributes for loading states
- **Motion Sensitivity**: Honors `prefers-reduced-motion` settings
- **Focus Management**: Non-interactive, doesn't interfere with navigation

### Implementation Details

```tsx
// Accessible skeleton with proper ARIA
<div role="status" aria-live="polite" aria-label="Loading content">
  <GlassSkeleton variant="shimmer" width={200} height={20} />
  <span className="sr-only">Loading, please wait...</span>
</div>
```

### Best Practices

- Always provide meaningful loading messages for screen readers
- Use appropriate `aria-live` regions for dynamic content
- Respect user motion preferences
- Ensure skeleton layouts match actual content structure
- Provide alternative loading indicators when needed

## Apple HIG Guidelines

### Visual Design

- **Clarity**: Clean, minimal loading indicators that match content structure
- **Deference**: Subtle animations that don't distract from the loading process
- **Depth**: Glassmorphism effects create visual hierarchy during loading

### Animation Principles

- **Smooth Transitions**: 60fps animations with hardware acceleration
- **Purposeful Motion**: Each variant serves a specific loading context
- **Respectful**: Honors system motion preferences

### Usage Guidelines

- **Default**: General content loading with subtle pulse
- **Shimmer**: Data fetching, progressive loading
- **Pulse**: Heartbeat-like loading for real-time content

### Layout Considerations

- **Consistent**: Skeleton layouts should match actual content
- **Proportional**: Maintain proper aspect ratios and spacing
- **Responsive**: Adapt skeleton layouts for different screen sizes

## Performance

### Optimization Features

- **CSS Animations**: Hardware-accelerated transforms
- **Minimal DOM**: Lightweight component structure
- **Memory Efficient**: No JavaScript animation loops
- **Responsive**: Adapts to different screen sizes

### Best Practices

- Use CSS animations instead of JavaScript for better performance
- Implement proper cleanup for skeleton states
- Consider skeleton complexity for large lists
- Use virtualization for very long skeleton lists

```tsx
// Performance-optimized skeleton list
const OptimizedSkeletonList = ({ count }) => {

  return (
    <div>
      {Array.from({ length: visibleCount }, (_, i) => (
        <SkeletonCard key={i} />
      ))}
    </div>
  );
};
```
