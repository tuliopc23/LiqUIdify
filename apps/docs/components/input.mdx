## Props

| Prop           | Type                                             | Default     | Description            |
| -------------- | ------------------------------------------------ | ----------- | ---------------------- |
| `type`         | `string`                                         | `"text"`    | HTML input type        |
| `size`         | `"sm" \| "md" \| "lg"`                           | `"md"`      | Input size             |
| `label`        | `string`                                         | -           | Input label            |
| `placeholder`  | `string`                                         | -           | Placeholder text       |
| `state`        | `"default" \| "success" \| "error" \| "warning"` | `"default"` | Validation state       |
| `message`      | `string`                                         | -           | Help or error message  |
| `leftIcon`     | `ReactNode`                                      | -           | Icon on the left side  |
| `rightIcon`    | `ReactNode`                                      | -           | Icon on the right side |
| `disabled`     | `boolean`                                        | `false`     | Disabled state         |
| `readOnly`     | `boolean`                                        | `false`     | Read-only state        |
| `required`     | `boolean`                                        | `false`     | Required field         |
| `value`        | `string`                                         | -           | Controlled value       |
| `defaultValue` | `string`                                         | -           | Default value          |
| `onChange`     | `(event) => void`                                | -           | Change event handler   |
| `onFocus`      | `(event) => void`                                | -           | Focus event handler    |
| `onBlur`       | `(event) => void`                                | -           | Blur event handler     |
| `className`    | `string`                                         | -           | Additional CSS classes |

## Usage Patterns

### Form Fields

```tsx
function ContactForm() {
  const [formData, setFormData] = useState({
    name: "",
    email: "",
    message: "",
  });

  const [errors, setErrors] = useState({});

  const handleSubmit = (e) => {
    e.preventDefault();
    // Validation logic
    const newErrors = {};
    if (!formData.name) newErrors.name = "Name is required";
    if (!formData.email) newErrors.email = "Email is required";

    setErrors(newErrors);

    if (Object.keys(newErrors).length === 0) {
      // Submit form
      console.log("Form submitted:", formData);
    }
  };

  return (
    <form onSubmit={handleSubmit} className="space-y-4">
      <GlassInput
        label="Full Name"
        placeholder="Enter your full name"
        value={formData.name}
        onChange={(e) =>
          setFormData((prev) => ({ ...prev, name: e.target.value }))
        }
        state={errors.name ? "error" : "default"}
        message={errors.name}
        required
      />

      <GlassInput
        type="email"
        label="Email Address"
        placeholder="Enter your email"
        value={formData.email}
        onChange={(e) =>
          setFormData((prev) => ({ ...prev, email: e.target.value }))
        }
        state={errors.email ? "error" : "default"}
        message={errors.email}
        required
      />

      <GlassButton type="submit" variant="primary">
        Submit
      </GlassButton>
    </form>
  );
}
```

### Search Input

```tsx
function SearchInput({ onSearch, placeholder = "Search..." }) {
  const [query, setQuery] = useState("");

  const handleClear = () => {
    setQuery("");
    onSearch("");
  };

  const handleChange = (e) => {
    const value = e.target.value;
    setQuery(value);
    onSearch(value);
  };

  return (
    <GlassInput
      placeholder={placeholder}
      value={query}
      onChange={handleChange}
      leftIcon={<Search className="w-4 h-4" />}
      rightIcon={
        query && (
          <button
            type="button"
            onClick={handleClear}
            className="p-1 hover:bg-gray-100 rounded"
          >
            <X className="w-3 h-3" />
          </button>
        )
      }
    />
  );
}
```

### Password Input with Visibility Toggle

```tsx
function PasswordInput({ label = "Password", ...props }) {
  const [showPassword, setShowPassword] = useState(false);

  return (
    <GlassInput
      type={showPassword ? "text" : "password"}
      label={label}
      leftIcon={<Lock className="w-4 h-4" />}
      rightIcon={
        <button
          type="button"
          onClick={() => setShowPassword(!showPassword)}
          className="p-1 hover:bg-gray-100 rounded"
        >
          {showPassword ? (
            <EyeOff className="w-4 h-4" />
          ) : (
            <Eye className="w-4 h-4" />
          )}
        </button>
      }
      {...props}
    />
  );
}
```

## Accessibility

The GlassInput component follows WCAG 2.1 guidelines:

### Keyboard Navigation

- **Tab**: Navigate between inputs
- **Enter**: Submit form (when in form)
- **Escape**: Clear input (when applicable)

### Screen Reader Support

- Proper label association
- Error message announcements
- State change notifications
- Required field indicators

### Visual Accessibility

- High contrast focus indicators
- Clear error states
- Sufficient color contrast
- Readable typography

### ARIA Attributes

- `aria-invalid` for error states
- `aria-describedby` for help text
- `aria-required` for required fields
- `aria-label` when no visible label

## Apple HIG Guidelines

Following Apple's Human Interface Guidelines for text fields:

### Visual Design

- **Clarity**: Clean, readable input styling
- **Depth**: Subtle glass effects create visual hierarchy
- **Consistency**: Uniform styling across states

### Interaction

- **Responsiveness**: Immediate visual feedback
- **Predictability**: Consistent behavior patterns
- **Feedback**: Clear validation states

### Content

- **Helpful**: Clear labels and placeholder text
- **Concise**: Brief, descriptive messaging
- **Contextual**: Appropriate for the input type

## Best Practices

### Labels and Placeholders

- Always provide clear labels
- Use placeholders for examples, not instructions
- Keep placeholder text concise
- Consider internationalization

### Validation

- Validate on blur for better UX
- Provide clear, actionable error messages
- Use appropriate validation states
- Consider real-time validation for complex fields

### Icons

- Use icons to clarify input purpose
- Ensure icons are accessible
- Don't rely solely on icons for meaning
- Keep icon usage consistent

### Responsive Design

- Ensure adequate touch targets
- Test on various screen sizes
- Consider mobile keyboard types
- Maintain readability at all sizes
