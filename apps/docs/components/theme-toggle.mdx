# ThemeToggle

A beautifully crafted theme toggle component that switches between light and dark modes with smooth animations and proper accessibility support. Follows Apple Human Interface Guidelines for intuitive user experience.

## Live Examples

```tsx

  return (
    <div className="flex items-center gap-4">
      <ThemeToggle />
      <span className="text-liquid-secondary">Current theme: {theme}</span>
    </div>
  );
}
```

## Theme Hook

The component relies on the `useTheme` hook which provides:

```typescript
interface ThemeProviderState {
  theme: "light" | "dark";
  setTheme: (theme: "light" | "dark") => void;
}
```

## Usage Patterns

### Basic Setup

```tsx
import "liquidify/styles";

function App() {
  return (
    <ThemeProvider defaultTheme="light" storageKey="my-app-theme">
      <div className="min-h-screen bg-glass-bg-canvas">
        <header className="p-4">
          <ThemeToggle />
        </header>
        {/* Your app content */}
      </div>
    </ThemeProvider>
  );
}
```

### Custom Theme Storage

```tsx
function App() {
  return (
    <ThemeProvider defaultTheme="dark" storageKey="custom-theme-key">
      {/* Your app */}
    </ThemeProvider>
  );
}
```

### Programmatic Theme Control

```tsx
import { initTheme, setTheme, getTheme } from "liquidify";

initTheme({ storageKey: "liquidify-theme" });

function CustomControls() {
  const theme = getTheme();
  return (
    <div className="flex gap-2">
      <button onClick={() => setTheme("light")} className={theme === "light" ? "active" : ""}>Light</button>
      <button onClick={() => setTheme("dark")} className={theme === "dark" ? "active" : ""}>Dark</button>
      <button onClick={() => setTheme(theme === "light" ? "dark" : "light")}>Toggle</button>
    </div>
  );
}
```

## Accessibility

The ThemeToggle component follows WCAG 2.1 AA guidelines:

### Keyboard Navigation

- **Tab**: Focus the toggle button
- **Space/Enter**: Activate theme switch
- **Escape**: Remove focus (standard behavior)

### Screen Reader Support

- Implicit `button` role with proper semantics
- Visual state changes are announced automatically
- Icon changes provide clear visual feedback

### ARIA Attributes

```tsx
// Enhanced version with explicit ARIA
function AccessibleThemeToggle() {
  const { theme, setTheme } = useTheme();

  return (
    <button
      type="button"
      onClick={() => setTheme(theme === "light" ? "dark" : "light")}
      aria-label={`Switch to ${theme === "light" ? "dark" : "light"} mode`}
      aria-pressed={theme === "dark"}
      className="glass-effect btn-scale rounded-lg p-2"
    >
      {theme === "light" ? (
        <Moon className="h-5 w-5" aria-hidden="true" />
      ) : (
        <Sun className="h-5 w-5" aria-hidden="true" />
      )}
    </button>
  );
}
```

## Apple Human Interface Guidelines

The ThemeToggle follows Apple's design principles:

### Visual Design

- **Clarity**: Clear sun/moon icons indicate current state
- **Deference**: Subtle glass effect doesn't compete with content
- **Depth**: Layered glass creates natural hierarchy

### Interaction Design

- **Predictable**: Standard toggle behavior users expect
- **Responsive**: Immediate visual feedback on interaction
- **Forgiving**: Large touch target (44pt minimum)

### Motion Design

- **Purposeful**: Smooth transitions enhance understanding
- **Respectful**: Honors `prefers-reduced-motion`
- **Natural**: Easing curves feel organic

### Color and Typography

- **Dynamic**: Adapts to system appearance preferences
- **Accessible**: Maintains contrast in both themes
- **Consistent**: Uses system color tokens

## Best Practices

### Do's

- ✅ Place in consistent location (usually top-right)
- ✅ Use with ThemeProvider for proper context
- ✅ Test with screen readers and keyboard navigation
- ✅ Respect user's system preference as default
- ✅ Persist theme choice across sessions

### Don'ts

- ❌ Don't override user's reduced motion preferences
- ❌ Don't place multiple theme toggles on same page
- ❌ Don't use without proper ThemeProvider setup
- ❌ Don't forget to test color contrast in both themes
- ❌ Don't make the toggle too small for touch devices
