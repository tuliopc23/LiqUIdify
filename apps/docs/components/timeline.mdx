## Props

| Prop          | Type                         | Default      | Description                                |
| ------------- | ---------------------------- | ------------ | ------------------------------------------ |
| `items`       | `TimelineItem[]`             | -            | Array of timeline items                    |
| `orientation` | `'vertical' \| 'horizontal'` | `'vertical'` | Timeline layout direction                  |
| `alternating` | `boolean`                    | `false`      | Alternate items left/right (vertical only) |
| `className`   | `string`                     | -            | Additional CSS classes                     |

### TimelineItem Interface

| Property      | Type                                   | Description                 |
| ------------- | -------------------------------------- | --------------------------- |
| `id`          | `string`                               | Unique identifier           |
| `title`       | `string`                               | Item title                  |
| `description` | `string`                               | Optional description        |
| `date`        | `string`                               | Optional date string        |
| `status`      | `'completed' \| 'active' \| 'pending'` | Item status                 |
| `icon`        | `ReactNode`                            | Optional custom icon        |
| `content`     | `ReactNode`                            | Optional additional content |

## Accessibility

- **Semantic Structure**: Uses proper heading hierarchy and list semantics
- **ARIA Labels**: Provides context for status indicators and icons
- **Keyboard Navigation**: Focusable elements follow logical tab order
- **Screen Reader**: Clear announcement of timeline structure and status
- **Color Independence**: Status conveyed through icons and text, not just color
- **High Contrast**: Maintains visibility in high contrast modes

## Apple HIG Guidelines

### Visual Design

- **Hierarchy**: Clear visual hierarchy with proper spacing and typography
- **Depth**: Glass effects and shadows create depth without overwhelming
- **Color**: Status colors follow system conventions (green=success, blue=active, blue=pending)
- **Typography**: Consistent font weights and sizes for readability

### Layout

- **Spacing**: Generous whitespace for easy scanning
- **Alignment**: Consistent alignment for professional appearance
- **Responsive**: Adapts gracefully to different screen sizes
- **Flow**: Natural reading flow from top to bottom or left to right

### Interaction

- **Feedback**: Subtle hover effects on interactive elements
- **States**: Clear visual states for different timeline statuses
- **Animation**: Smooth transitions when content changes
- **Touch**: Appropriate touch targets for mobile devices

### Content Guidelines

- **Clarity**: Use clear, concise titles and descriptions
- **Consistency**: Maintain consistent date formats and terminology
- **Context**: Provide enough context for each timeline item
- **Scannable**: Structure content for easy scanning

### Best Practices

- Use consistent date formatting throughout
- Provide meaningful status indicators
- Consider content length for responsive design
- Test with various content lengths
- Ensure proper contrast for all status colors
- Use icons consistently to reinforce meaning
