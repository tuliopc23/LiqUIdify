## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `placeholder` | `string` | `'Search...'` | Input placeholder text |
| `suggestions` | `SearchSuggestion[]` | `[]` | Array of search suggestions |
| `recentSearches` | `string[]` | `[]` | Array of recent search terms |
| `onSearch` | `(query: string) => void` | - | Callback when search is performed |
| `onSuggestionClick` | `(suggestion: SearchSuggestion) => void` | - | Callback when suggestion is clicked |
| `showTrending` | `boolean` | `false` | Show trending indicators |
| `maxSuggestions` | `number` | `8` | Maximum suggestions to display |
| `className` | `string` | - | Additional CSS classes |
| `id` | `string` | - | Input element ID |

### SearchSuggestion Interface

| Property | Type | Description |
|----------|------|-------------|
| `id` | `string` | Unique suggestion identifier |
| `text` | `string` | Suggestion display text |
| `type` | `'recent' \| 'suggestion' \| 'trending'` | Suggestion type |
| `category` | `string` | Optional category label |
| `count` | `number` | Optional popularity count |

## Usage Patterns

### Documentation Search

```tsx
import { GlassSearch } from '@liquidify/components'
import { useRouter } from 'next/router'

function DocumentationSearch() {
  const router = useRouter()
  const [suggestions] = useState([
    { id: '1', text: 'Getting Started', type: 'suggestion', category: 'Guides' },
    { id: '2', text: 'Component API', type: 'suggestion', category: 'Reference' },
    { id: '3', text: 'Theming', type: 'trending', count: 1250 },
  ])
  
  const handleSearch = (query) => {
    router.push(`/search?q=${encodeURIComponent(query)}`)
  }
  
  const handleSuggestionClick = (suggestion) => {
    // Navigate to specific page based on suggestion
    if (suggestion.category === 'Guides') {
      router.push(`/guides/${suggestion.text.toLowerCase().replace(' ', '-')}`)
    } else {
      handleSearch(suggestion.text)
    }
  }
  
  return (
    <GlassSearch 
      placeholder="Search documentation..."
      suggestions={suggestions}
      onSearch={handleSearch}
      onSuggestionClick={handleSuggestionClick}
    />
  )
}
```

```tsx
        isOpen={isOpen}
        onClose={() => setIsOpen(false)}

    </>
  )
}
```

## Accessibility

### WCAG Compliance

- **Keyboard Navigation**: Full keyboard support with arrow keys
- **Screen Reader**: Proper ARIA attributes and live regions
- **Focus Management**: Clear focus indicators and logical tab order
- **Search Results**: Announced to screen readers

### Keyboard Shortcuts

| Key | Action |
|-----|--------|
| `Enter` | Perform search with current query |
| `Arrow Down` | Navigate to next suggestion |
| `Arrow Up` | Navigate to previous suggestion |
| `Escape` | Close suggestions and clear focus |
| `Tab` | Move focus away from search |

## Apple HIG Guidelines

### Visual Design

- **Clarity**: Clear visual hierarchy in search results
- **Depth**: Subtle glass effects that enhance the search metaphor
- **Consistency**: Consistent styling across suggestion types
- **Feedback**: Immediate visual feedback for user interactions

### Interaction Design

- **Immediate Response**: Instant suggestion display as user types
- **Predictive**: Smart suggestions based on context and history
- **Efficient**: Quick access to recent and trending searches
- **Forgiving**: Handles typos and partial matches gracefully

### Content Organization

- **Categorization**: Logical grouping of suggestions by type
- **Prioritization**: Most relevant suggestions appear first
- **Context**: Additional information (categories, counts) when helpful
- **Scannability**: Easy to scan and compare options

## Best Practices

### Do's

- ✅ Provide relevant, contextual suggestions
- ✅ Show recent searches for quick access
- ✅ Use clear, descriptive placeholder text
- ✅ Handle empty states gracefully
- ✅ Implement proper keyboard navigation

### Don'ts

- ❌ Don't overwhelm with too many suggestions
- ❌ Don't show irrelevant or outdated suggestions
- ❌ Don't ignore keyboard accessibility
- ❌ Don't make the search box too small
- ❌ Don't forget to handle loading states

### Performance

- **Debounced Input**: Prevent excessive API calls
- **Cached Results**: Store recent searches and suggestions
- **Lazy Loading**: Load suggestions on demand
- **Efficient Filtering**: Client-side filtering for better performance

```tsx
// Debounced search implementation
import { useDebouncedCallback } from 'use-debounce'

const debouncedSearch = useDebouncedCallback(
  async (query) => {
    const results = await searchAPI(query)
    setSuggestions(results)
  },
  300
)

const handleInputChange = (value) => {
  setQuery(value)
  if (value.length > 2) {
    debouncedSearch(value)
  }
}
```

### Analytics Integration

```tsx
// Track search analytics
const handleSearch = (query) => {
  analytics.track('search_performed', {
    query,
    timestamp: Date.now(),
    source: 'main_search'
  })

const handleSuggestionClick = (suggestion) => {
  analytics.track('suggestion_selected', {
    suggestion: suggestion.text,
    type: suggestion.type,
    position: suggestions.indexOf(suggestion)
  })

}
```
