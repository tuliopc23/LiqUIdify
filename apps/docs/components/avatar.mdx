# Avatar

A glassmorphic avatar component for displaying user profile images with elegant fallbacks, status indicators, and multiple size variants. Features Apple-inspired design with smooth glass effects.

## Props

| Prop         | Type                                            | Default      | Description                      |
| ------------ | ----------------------------------------------- | ------------ | -------------------------------- |
| `src`        | `string`                                        | -            | Image source URL                 |
| `alt`        | `string`                                        | -            | Alt text for the image           |
| `size`       | `"xs" \| "sm" \| "md" \| "lg" \| "xl" \| "2xl"` | `"md"`       | Avatar size                      |
| `variant`    | `"circular" \| "rounded" \| "square"`           | `"circular"` | Avatar shape                     |
| `fallback`   | `string`                                        | -            | Fallback text (usually initials) |
| `showBorder` | `boolean`                                       | `false`      | Show glass border effect         |
| `status`     | `"online" \| "offline" \| "away" \| "busy"`     | -            | Status indicator                 |
| `className`  | `string`                                        | -            | Additional CSS classes           |

## Usage Patterns

### User Profile

```tsx
function UserProfile({ user }) {
  return (
    <div className="flex items-center gap-3">
      <GlassAvatar
        src={user.avatar}
        alt={user.name}
        fallback={user.initials}
        status={user.status}
        size="lg"
        showBorder
      />
      <div>
        <h3 className="font-semibold">{user.name}</h3>
        <p className="text-sm text-[var(--apple-text-secondary)]">
          {user.role}
        </p>
      </div>
    </div>
  );
}
```

### Avatar Group

```tsx
function AvatarGroup({ users, max = 3 }) {
  const displayUsers = users.slice(0, max);
  const remainingCount = users.length - max;

  return (
    <div className="flex -space-x-2">
      {displayUsers.map((user, index) => (
        <GlassAvatar
          key={user.id}
          src={user.avatar}
          alt={user.name}
          fallback={user.initials}
          showBorder
          className="relative z-10"
          style={{ zIndex: displayUsers.length - index }}
        />
      ))}
      {remainingCount > 0 && (
        <GlassAvatar
          fallback={`+${remainingCount}`}
          showBorder
          className="relative z-0"
        />
      )}
    </div>
  );
}
```

## Accessibility

The GlassAvatar component follows WCAG 2.1 guidelines:

### Image Accessibility

- Proper `alt` text for profile images
- Fallback content for failed image loads
- Meaningful initials when no image available

- Color is not the only indicator of status
- Status information available to screen readers
- Sufficient contrast for status indicators

### Focus Management

- Focusable when interactive (clickable)
- Clear focus indicators
- Logical tab order in groups

## Apple HIG Guidelines

Following Apple's Human Interface Guidelines for user representation:

### Visual Design

- **Clarity**: Clean, recognizable user representation
- **Depth**: Subtle glass effects create visual depth
- **Consistency**: Uniform sizing and styling

### Content

- **Personal**: Use actual user photos when available
- **Fallback**: Meaningful initials or icons
- **Respectful**: Appropriate image handling and privacy

### Interaction

- **Responsive**: Immediate feedback on interaction
- **Contextual**: Appropriate for the use case
- **Accessible**: Works for all users

## Best Practices

### Image Handling

- Provide meaningful alt text
- Use appropriate image sizes (avoid large images)
- Handle loading states gracefully
- Implement error fallbacks

### Fallback Content

- Use user initials when possible
- Ensure initials are readable
- Consider cultural naming conventions
- Provide generic fallback for edge cases

- Use consistent status meanings
- Don't rely solely on color
- Update status in real-time when possible
- Consider privacy implications

### Performance

- Optimize image sizes
- Use appropriate image formats
- Consider lazy loading for lists
- Cache images when possible
