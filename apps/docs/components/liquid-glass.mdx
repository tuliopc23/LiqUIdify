# LiquidGlass System

The LiquidGlass system is the foundation of LiqUIdify's design language, providing Apple-inspired glassmorphism effects with layered transparency, specular highlights, and content-aware adaptation.

## Provider Configuration

### GlassUIProvider + LiquidGlass Config

```typescript
interface LiquidGlassConfig {
  /** Base color as RGB string (e.g., "255,255,255") */
  color?: string;
  /** Blur amount in pixels */
  blur?: number;
  /** Saturation percentage */
  saturation?: number;
  /** Opacity (0-1) */
  opacity?: number;
  /** Enable content-aware adaptation */
  adaptToContent?: boolean;
  /** Enable specular highlights */
  specularHighlights?: boolean;
  /** Enable magnetic hover effects */
  magneticHover?: boolean;
}

interface GlassUIProviderProps {
  children: React.ReactNode;
  glassConfig?: LiquidGlassConfig;
}
```

### Default Configuration

```typescript
const defaultConfig: Required<LiquidGlassConfig> = {
  color: "255,255,255",
  blur: 24,
  saturation: 180,
  opacity: 0.75,
  adaptToContent: true,
  specularHighlights: true,
  magneticHover: false,
};
```

## Usage Patterns

### Basic Setup

```tsx
import "liquidify/styles";

function App() {
  return (
    <GlassUIProvider
      glassConfig={{
        blur: 24,
        opacity: 0.75,
        adaptToContent: true,
      }}
    >
      {/* Your app content */}
    </GlassUIProvider>
  );
}
```

### Custom Glass Component

```tsx
import { forwardRef } from "react";
import { cn } from "liquidify/utils";

interface CustomGlassProps {
  children: React.ReactNode;
  size?: "sm" | "md" | "lg";
  interactive?: boolean;
  className?: string;
}

const CustomGlass = forwardRef<HTMLDivElement, CustomGlassProps>(
  (
    { children, size = "md", interactive = false, className, ...props },
    ref,
  ) => {
    return (
      <div
        ref={ref}
        className={cn(
          "liquid-glass-container",
          `liquid-glass-${size}`,
          { "liquid-glass-interactive": interactive },
          className,
        )}
        {...props}
      >
        <div className="liquid-glass-filter" />
        <div className="liquid-glass-overlay" />
        <div className="liquid-glass-specular" />
        <div className="liquid-glass-content">{children}</div>
      </div>
    );
  },
);
```

### Content-Aware Hook

```tsx
import { useRef, useEffect } from "react";

function ContentAwareComponent() {
  const contentRef = useRef<HTMLDivElement>(null);
  const { updateGlassStyle } = useLiquidGlass();

  useEffect(() => {
    // Analyze content and update glass style
    if (contentRef.current) {
      const analysis = analyzeContent(contentRef.current);
      updateGlassStyle(analysis);
    }
  }, []);

  return (
    <div ref={contentRef} className="liquid-glass-container">
      {/* Content that affects glass appearance */}
    </div>
  );
}
```

## Accessibility

The LiquidGlass system follows WCAG 2.1 AA guidelines:

### Visual Accessibility

- **Contrast**: Maintains 4.5:1 contrast ratio for text
- **Transparency**: Never obscures important content
- **Focus**: Clear focus indicators on interactive elements

### Motion Accessibility

```css
/* Respects reduced motion preferences */
@media (prefers-reduced-motion: reduce) {
  .liquid-glass-container,
  .liquid-glass-interactive {
    transition: none !important;
    animation: none !important;
  }
}
```

### High Contrast Mode

```css
/* Adapts to high contrast preferences */
@media (prefers-contrast: more) {
  .liquid-glass-container {
    backdrop-filter: none;
    background: rgba(255, 255, 255, 0.95);
    border: 2px solid rgba(0, 0, 0, 0.8);
  }
}
```

### Screen Reader Support

- Glass effects are purely visual and don't interfere with content
- Proper semantic markup maintained within glass containers
- Interactive elements have appropriate ARIA attributes

## Apple Human Interface Guidelines

The LiquidGlass system embodies Apple's design principles:

### Clarity

- **Legible**: Text remains readable over glass effects
- **Focused**: Glass enhances rather than distracts from content
- **Purposeful**: Every layer serves a visual function

### Deference

- **Subtle**: Glass effects don't compete with content
- **Adaptive**: Responds to content and context
- **Respectful**: Honors user preferences and accessibility needs

### Depth

- **Layered**: Multiple layers create realistic depth
- **Hierarchical**: Visual hierarchy through glass intensity
- **Spatial**: Clear relationships between elements

### Material Design

- **Authentic**: Behaves like real glass material
- **Physical**: Responds to light and environment
- **Consistent**: Predictable behavior across components

- CSS custom properties for runtime changes
- Minimal JavaScript for content analysis
- Debounced resize and mutation observers
- Optimized backdrop-filter usage

### Bundle Size

- Core CSS: ~8KB gzipped
- JavaScript hooks: ~2KB gzipped
- Tree-shakeable imports
- No runtime dependencies

## Best Practices

### Do's

- ✅ Use consistent glass hierarchy throughout app
- ✅ Test with various background content
- ✅ Ensure sufficient contrast for text
- ✅ Respect user motion preferences
- ✅ Use semantic HTML within glass containers

### Don'ts

- ❌ Don't stack too many glass layers
- ❌ Don't use glass for critical UI elements only
- ❌ Don't ignore accessibility requirements
- ❌ Don't override user system preferences
- ❌ Don't use glass effects that impair readability
