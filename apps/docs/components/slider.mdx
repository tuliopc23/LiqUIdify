import { Slider } from "liquidify";

### Example

```tsx
import { Slider } from "liquidify";

export default function Demo() {
  return (
    <div>
      <h3>Basic Slider</h3>
      <Slider label="Basic" min={0} max={100} />

      <h3>Volume Control</h3>
      <Slider label="Volume" min={0} max={100} step={5} />

      <h3>Price Range ($100-$500)</h3>
      <Slider label="Price" min={100} max={500} step={10} />
    </div>
  );
}
```

## Props

| Prop        | Type                         | Default | Description                           |
| ----------- | ---------------------------- | ------- | ------------------------------------- |
| `label`     | `string`                     | -       | Optional label displayed above track  |
| `size`      | `'sm' \| 'md' \| 'lg'`      | `'md'`  | Control size                          |
| `min`       | `number`                     | `0`     | Minimum value                         |
| `max`       | `number`                     | `100`   | Maximum value                         |
| `step`      | `number`                     | `1`     | Step increment                        |
| `disabled`  | `boolean`                    | `false` | Whether slider is disabled            |
| `className` | `string`                     | -       | Additional className if needed        |
| ...rest     | Slider root props (from Ark) | -       | You can pass any Ark Slider props     |

## Usage Patterns

### Volume Control

```tsx
<Slider label="Volume" min={0} max={100} step={5} />
```

### Price Range

```tsx
<Slider label="Price" min={100} max={500} step={10} />
```

### Disabled State

```tsx
<Slider label="Disabled" min={0} max={100} disabled />
```

## Accessibility

- **ARIA Support**: Implements proper ARIA attributes for screen readers
- **Keyboard Navigation**:
  - Arrow keys to adjust value
  - Home/End keys to jump to min/max
  - Page Up/Down for larger increments
- **Focus Management**: Clear focus indicators and proper tab order
- **Screen Reader**: Announces current value and range information
- **High Contrast**: Maintains visibility in high contrast modes

## Apple HIG Guidelines

### Visual Design

- **Depth**: Uses subtle shadows and glass effects for depth perception
- **Motion**: Smooth animations with spring physics for natural feel
- **Color**: Follows system color schemes with proper contrast ratios
- **Typography**: Clear value display with appropriate font weights

### Interaction

- **Touch Targets**: Minimum 44pt touch target for thumb control
- **Feedback**: Immediate visual feedback on interaction
- **Gestures**: Supports both click/drag and keyboard interaction
- **States**: Clear visual states for normal, hover, active, and disabled

### Best Practices

- Use appropriate step values for the data type
- Provide clear labels and value indicators
- Consider the range and precision needed for your use case
- Test with assistive technologies
- Ensure sufficient color contrast for accessibility
