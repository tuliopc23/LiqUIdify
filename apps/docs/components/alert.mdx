# Alert

A glassmorphic alert component for displaying important messages with semantic color variants. Perfect for notifications, warnings, and status updates with Apple-inspired design.

## Props

| Prop        | Type                                          | Default   | Description                                  |
| ----------- | --------------------------------------------- | --------- | -------------------------------------------- |
| `variant`   | `"info" \| "success" \| "warning" \| "error"` | `"info"`  | Semantic variant affecting color and styling |
| `title`     | `string`                                      | -         | Optional title displayed prominently         |
| `children`  | `ReactNode`                                   | -         | **Required.** Alert content                  |
| `className` | `string`                                      | -         | Additional CSS classes                       |
| `role`      | `string`                                      | `"alert"` | ARIA role for accessibility                  |

## Usage Patterns

### Form Validation

```tsx
import { useState } from "react";

function FormWithValidation() {
  const [error, setError] = useState("");
  const [success, setSuccess] = useState(false);

  const handleSubmit = (e) => {
    e.preventDefault();
    const formData = new FormData(e.target);
    const email = formData.get("email");

    if (!email) {
      setError("Email is required");
      return;
    }

    setError("");
    setSuccess(true);
  };

  return (
    <form onSubmit={handleSubmit} className="space-y-4">
      {error && (
        <GlassAlert variant="error" title="Validation Error">
          {error}
        </GlassAlert>
      )}

      {success && (
        <GlassAlert variant="success" title="Success">
          Form submitted successfully!
        </GlassAlert>
      )}

      <GlassInput name="email" type="email" placeholder="Email" />
      <GlassButton type="submit">Submit</GlassButton>
    </form>
  );
}
```

### Status Messages

```tsx
function StatusMessages({ status }) {
  const alerts = {
    loading: (
      <GlassAlert variant="info" title="Loading">
        Please wait while we process your request...
      </GlassAlert>
    ),
    success: (
      <GlassAlert variant="success" title="Complete">
        Your data has been saved successfully.
      </GlassAlert>
    ),
    error: (
      <GlassAlert variant="error" title="Failed">
        Unable to save your data. Please try again.
      </GlassAlert>
    ),
  };

  return alerts[status] || null;
}
```

## Accessibility

The GlassAlert component follows WCAG 2.1 guidelines:

### ARIA Support

- Default `role="alert"` for important messages
- Proper semantic structure with headings
- Screen reader announcements for dynamic content

### Visual Design

- High contrast colors for readability
- Clear visual hierarchy with titles
- Sufficient color contrast ratios
- Icons complement text (not sole indicators)

### Content Guidelines

- Clear, concise messaging
- Actionable information when possible
- Avoid jargon or technical terms
- Include next steps for errors

## Apple HIG Guidelines

Following Apple's Human Interface Guidelines for alerts and notifications:

### Visual Design

- **Clarity**: Clean typography and clear visual hierarchy
- **Depth**: Subtle glassmorphism effects create depth
- **Color**: Semantic colors that convey meaning

### Content

- **Concise**: Brief, scannable messages
- **Helpful**: Actionable information and next steps
- **Tone**: Appropriate tone for the message type

### Interaction

- **Non-intrusive**: Doesn't block user workflow
- **Contextual**: Appears near relevant content
- **Dismissible**: Can be easily dismissed when appropriate

## Best Practices

### Message Content

- Keep messages concise and scannable
- Use clear, non-technical language
- Provide actionable next steps
- Include relevant context

### Visual Hierarchy

- Use titles for important alerts
- Group related information
- Maintain consistent spacing
- Consider message priority

### Timing

- Show alerts at appropriate moments
- Don't overwhelm users with multiple alerts
- Consider auto-dismissal for success messages
- Keep error messages visible until resolved

### Responsive Design

- Ensure readability on all screen sizes
- Stack content appropriately on mobile
- Maintain touch-friendly spacing
- Test with various content lengths

## Examples

### API Error Handling

```tsx
function APIErrorAlert({ error }) {
  const getErrorMessage = (error) => {
    switch (error.status) {
      case 401:
        return {
          title: "Authentication Required",
          message: "Please log in to continue.",
        };
      case 403:
        return {
          title: "Access Denied",
          message: "You don't have permission to perform this action.",
        };
      case 500:
        return {
          title: "Server Error",
          message: "Something went wrong on our end. Please try again later.",
        };
      default:
        return {
          title: "Error",
          message: "An unexpected error occurred.",
        };
    }
  };

  const { title, message } = getErrorMessage(error);

  return (
    <GlassAlert variant="error" title={title}>
      {message}
    </GlassAlert>
  );
}
```
