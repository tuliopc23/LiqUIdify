# Button

A versatile glassmorphic button component with liquid glass effects, multiple variants, sizes, and comprehensive interaction states. Features Apple-inspired design with smooth animations and full accessibility support.

### Loading States

```tsx live
import { useState } from "react";
import { GlassButton } from "@liquidify/components";
import { Save } from "lucide-react";

export default function Demo() {
  const [loading, setLoading] = useState(false);

  const handleClick = () => {
    setLoading(true);
    setTimeout(() => setLoading(false), 2000);
  };

  return (
    <div className="flex items-center gap-4 flex-wrap">
      <GlassButton loading>Loading...</GlassButton>

      <GlassButton loading loadingText="Saving...">
        Save
      </GlassButton>

      <GlassButton
        loading={loading}
        loadingText="Processing..."
        leftIcon={<Save className="w-4 h-4" />}
        onClick={handleClick}
      >
        {loading ? "Processing..." : "Process Data"}
      </GlassButton>

      <GlassButton loading iconOnly>
        <Save className="w-4 h-4" />
      </GlassButton>
    </div>
  );
}
```

## Props

| Prop          | Type                                                   | Default     | Description                            |
| ------------- | ------------------------------------------------------ | ----------- | -------------------------------------- |
| `variant`     | `"primary" \| "secondary" \| "ghost" \| "destructive"` | `"primary"` | Button style variant                   |
| `size`        | `"sm" \| "md" \| "lg" \| "xl"`                         | `"md"`      | Button size                            |
| `leftIcon`    | `ReactNode`                                            | -           | Icon displayed on the left side        |
| `rightIcon`   | `ReactNode`                                            | -           | Icon displayed on the right side       |
| `loading`     | `boolean`                                              | `false`     | Show loading state with spinner        |
| `loadingText` | `string`                                               | -           | Text to show during loading state      |
| `fullWidth`   | `boolean`                                              | `false`     | Make button full width                 |
| `iconOnly`    | `boolean`                                              | `false`     | Icon-only button (square aspect ratio) |
| `asChild`     | `boolean`                                              | `false`     | Render as child component              |
| `disabled`    | `boolean`                                              | `false`     | Disabled state                         |
| `children`    | `ReactNode`                                            | -           | Button content                         |
| `className`   | `string`                                               | -           | Additional CSS classes                 |
| `onClick`     | `(event) => void`                                      | -           | Click event handler                    |

## Usage Patterns

### Form Actions

```tsx

import { GlassButton, GlassInput } from "@liquidify/components";
import { Save, X } from "lucide-react";

function FormActions({ onSave, onCancel, isLoading }) {
  return (
    <div className="flex justify-end gap-3">

        loadingText="Saving..."

      </GlassButton>
    </div>
  );
}
```

## Accessibility

The GlassButton component follows WCAG 2.1 guidelines:

### Keyboard Navigation

- **Tab**: Navigate between buttons
- **Enter/Space**: Activate button
- **Escape**: Cancel action (context-dependent)

### Screen Reader Support

- Proper button semantics
- Loading state announcements
- Disabled state handling
- Icon descriptions when needed

### Focus Management

- Clear focus indicators
- Logical tab order
- Focus trapping in modals
- Appropriate focus restoration

### Visual Accessibility

- High contrast ratios
- Clear visual hierarchy
- Sufficient touch targets (44px minimum)
- Color is not the sole indicator

## Apple HIG Guidelines

Following Apple's Human Interface Guidelines for buttons:

### Visual Design

- **Clarity**: Clear, readable button labels
- **Depth**: Subtle glass effects create visual hierarchy
- **Consistency**: Uniform styling across variants

### Interaction

- **Responsiveness**: Immediate visual feedback
- **Predictability**: Consistent behavior patterns
- **Feedback**: Clear loading and disabled states

### Content

- **Concise**: Brief, action-oriented labels
- **Descriptive**: Clear purpose and outcome
- **Consistent**: Uniform terminology

## Best Practices

### Button Labels

- Use action verbs ("Save", "Delete", "Continue")
- Keep labels concise and descriptive
- Avoid generic terms like "Click here"
- Consider internationalization

### Visual Hierarchy

- Use primary buttons for main actions
- Limit primary buttons per view
- Group related actions together
- Maintain consistent spacing

### Loading States

- Show loading for actions taking >500ms
- Provide meaningful loading text
- Disable button during loading
- Consider progress indicators for long operations

### Responsive Design

- Ensure adequate touch targets
- Stack buttons on mobile when needed
- Consider full-width on small screens
- Test with various content lengths

function ProductActions({ product, onAddToCart, onBuyNow, isLoading }) {

### File Upload

```tsx
function FileUpload({ onUpload, isUploading, progress }) {
  return (
    <div className="text-center p-8 border-2 border-dashed border-gray-300 rounded-lg">
      <div className="space-y-4">
        <div className="text-gray-600">
          {isUploading
            ? `Uploading... ${progress}%`

        </div>

        <GlassButton
          variant="primary"
          loading={isUploading}
          loadingText={`Uploading... ${progress}%`}

          leftIcon={!isUploading && <Upload className="w-4 h-4" />}
        >
          {isUploading ? `${progress}%` : "Choose Files"}
        </GlassButton>
      </div>
    </div>
  );
}
```
