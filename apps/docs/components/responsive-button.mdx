# Responsive Button

The `GlassResponsiveButton` component provides an enhanced button experience with responsive sizing, adaptive touch interactions, and glassmorphism effects. It automatically adjusts for mobile devices and provides optimal touch targets.

### Loading States

### Loading States

```tsx
export default function Demo() {
  const [loading1, setLoading1] = useState(false);
  const [loading2, setLoading2] = useState(false);

  const handleClick1 = () => {
    setLoading1(true);
    setTimeout(() => setLoading1(false), 2000);
  };

  const handleClick2 = () => {
    setLoading2(true);
    setTimeout(() => setLoading2(false), 3000);
  };

  return (
    <div style={{ display: "flex", gap: 12, flexWrap: "wrap", padding: 20 }}>
      <GlassResponsiveButton
        loading={loading1}
        onClick={handleClick1}
        leftIcon={<Save />}
      >
        {loading1 ? "Saving..." : "Save Document"}
      </GlassResponsiveButton>

      <GlassResponsiveButton
        loading={loading2}
        onClick={handleClick2}
        variant="secondary"
        rightIcon={<Send />}
      >
        {loading2 ? "Sending..." : "Send Message"}
      </GlassResponsiveButton>
    </div>
  );
}
```

### Touch Interaction Demo

### Touch Interaction Demo

```tsx
export default function Demo() {
  const [touchCount, setTouchCount] = useState(0);
  const [lastTouch, setLastTouch] = useState(null);

  const handleTouch = () => {
    setTouchCount((prev) => prev + 1);
    setLastTouch(new Date().toLocaleTimeString());
  };

  return (
    <div style={{ padding: 20, textAlign: "center" }}>
      <div style={{ marginBottom: 20 }}>
        <h4>Touch Interaction Test</h4>
        <p style={{ fontSize: "0.875rem", opacity: 0.7 }}>
          Touch count: {touchCount}
        </p>
        {lastTouch && (
          <p style={{ fontSize: "0.75rem", opacity: 0.5 }}>
            Last touch: {lastTouch}
          </p>
        )}
      </div>

      <GlassResponsiveButton
        size="xl"
        variant="primary"
        onClick={handleTouch}
        leftIcon={<Touch />}
        fullWidth
      >
        Touch Me! (Optimized for Mobile)
      </GlassResponsiveButton>

      <div style={{ marginTop: 16 }}>
        <GlassResponsiveButton
          size="sm"
          variant="ghost"
          onClick={() => {
            setTouchCount(0);
            setLastTouch(null);
          }}
        >
          Reset Counter
        </GlassResponsiveButton>
      </div>
    </div>
  );
}
```

## Props

| Prop         | Type                                                                 | Default     | Description                 |
| ------------ | -------------------------------------------------------------------- | ----------- | --------------------------- |
| `variant`    | `"primary" \| "secondary" \| "tertiary" \| "ghost" \| "destructive"` | `"primary"` | Visual style variant        |
| `size`       | `"xs" \| "sm" \| "md" \| "lg" \| "xl"`                               | `"md"`      | Button size                 |
| `leftIcon`   | `React.ReactNode`                                                    | `undefined` | Icon displayed on the left  |
| `rightIcon`  | `React.ReactNode`                                                    | `undefined` | Icon displayed on the right |
| `loading`    | `boolean`                                                            | `false`     | Show loading state          |
| `fullWidth`  | `boolean`                                                            | `false`     | Make button full width      |
| `responsive` | `boolean`                                                            | `true`      | Enable responsive sizing    |
| `asChild`    | `boolean`                                                            | `false`     | Render as child component   |
| `disabled`   | `boolean`                                                            | `false`     | Disable the button          |
| `className`  | `string`                                                             | `undefined` | Additional CSS classes      |

## Accessibility

### WCAG Compliance

- **WCAG 2.1 AA**: Full compliance with touch targets and keyboard navigation
- **Touch Targets**: Minimum 44px touch targets on mobile
- **Keyboard Navigation**: Full keyboard support with focus indicators
- **Screen Readers**: Proper ARIA attributes and state announcements

### Implementation Details

```tsx
// Accessible responsive button
<GlassResponsiveButton
  aria-label="Save document"
  aria-describedby="save-help"
  aria-pressed={isSaved}
  disabled={isDisabled}
  loading={isLoading}
>
  Save
</GlassResponsiveButton>

<div id="save-help" className="sr-only">
  Saves the current document to your account
</div>
```

### Best Practices

- Ensure minimum 44px touch targets on mobile devices
- Provide clear focus indicators for keyboard navigation
- Use meaningful labels and descriptions
- Handle loading states appropriately for screen readers
- Include proper ARIA attributes for button states

## Apple HIG Guidelines

### Visual Design

- **Clarity**: Clean, minimal button design with clear hierarchy
- **Deference**: Glassmorphism enhances without overwhelming content
- **Depth**: Layered effects create clear interaction affordances

### Touch Interaction

- **Responsive**: Immediate visual feedback for all interactions
- **Intuitive**: Familiar button behaviors and animations
- **Accessible**: Optimal touch targets for all users

### Size Guidelines

- **Extra Small (xs)**: Compact interfaces, secondary actions
- **Small (sm)**: Toolbar buttons, inline actions
- **Medium (md)**: Default for most use cases
- **Large (lg)**: Primary actions, important buttons
- **Extra Large (xl)**: Call-to-action buttons, hero sections

### Responsive Behavior

- **Mobile First**: Larger touch targets on mobile devices
- **Adaptive**: Automatically adjusts for device capabilities
- **Consistent**: Maintains visual hierarchy across screen sizes

## Performance

### Optimization Features

- **Hardware Acceleration**: CSS transforms for smooth animations
- **Touch Optimization**: Optimized touch event handling
- **Responsive Detection**: Efficient device capability detection
- **Memory Efficient**: Minimal JavaScript overhead

### Best Practices

- Use CSS transforms for animations instead of changing layout properties
- Implement proper event delegation for touch interactions
- Consider button complexity for performance-critical interfaces
- Use `will-change` property sparingly for animation optimization

```tsx
// Performance-optimized button
const OptimizedButton = React.memo(({ children, ...props }) => (
  <GlassResponsiveButton {...props}>{children}</GlassResponsiveButton>
));
```
