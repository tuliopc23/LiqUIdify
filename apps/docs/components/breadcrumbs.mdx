## Props

| Prop          | Type                              | Default            | Description                                  |
| ------------- | --------------------------------- | ------------------ | -------------------------------------------- |
| `items`       | `BreadcrumbItem[]`                | -                  | Array of breadcrumb items                    |
| `separator`   | `ReactNode`                       | `<ChevronRight />` | Custom separator element                     |
| `showHome`    | `boolean`                         | `true`             | Whether to show home icon                    |
| `onHomeClick` | `() => void`                      | -                  | Callback when home icon is clicked           |
| `maxItems`    | `number`                          | -                  | Maximum items to show (truncates with "...") |
| `variant`     | `'default' \| 'solid' \| 'ghost'` | `'default'`        | Visual style variant                         |
| `size`        | `'sm' \| 'md' \| 'lg'`            | `'md'`             | Size variant                                 |
| `className`   | `string`                          | -                  | Additional CSS classes                       |

### BreadcrumbItem Interface

| Property  | Type         | Description                 |
| --------- | ------------ | --------------------------- |
| `label`   | `string`     | Display text                |
| `href`    | `string`     | Optional URL for navigation |
| `onClick` | `() => void` | Optional click handler      |
| `icon`    | `ReactNode`  | Optional icon element       |

## Usage Patterns

### E-commerce Navigation

```tsx
const productBreadcrumbs = [
  {
    label: 'Electronics',
    onClick: () => navigate('/electronics')
  },
  {
    label: 'Smartphones',
    onClick: () => navigate('/electronics/smartphones')
  },
  {
    label: 'iPhone 15 Pro'
  }
]

<GlassBreadcrumbs
  items={productBreadcrumbs}
  onHomeClick={() => navigate('/')}
/>
```

### Admin Dashboard

```tsx


const adminBreadcrumbs = [
  {
    label: 'Users',
    icon: <Users className="h-4 w-4" />,
    onClick: () => navigate('/admin/users')
  },
  {
    label: 'John Doe',
    icon: <User className="h-4 w-4" />,
    onClick: () => navigate('/admin/users/123')
  },
  {
    label: 'Edit Profile',
    icon: <Edit className="h-4 w-4" />
  }
]

<GlassBreadcrumbs
  items={adminBreadcrumbs}
  variant="solid"
  size="lg"
/>
```

### File System Navigation

```tsx
const pathBreadcrumbs = currentPath.split('/').map((segment, index, array) => ({
  label: segment || 'Root',
  onClick: index < array.length - 1
    ? () => navigateToPath(array.slice(0, index + 1).join('/'))
    : undefined
}))

<GlassBreadcrumbs
  items={pathBreadcrumbs}
  separator={<Slash className="h-4 w-4" />}
  maxItems={5}
/>
```

### Documentation Navigation

```tsx
<GlassBreadcrumbs
  items={[
    { label: "Docs", onClick: () => navigate("/docs") },
    { label: "Components", onClick: () => navigate("/docs/components") },
    { label: "Breadcrumbs" },
  ]}
  variant="ghost"
  showHome={false}
/>
```

## Accessibility

- **ARIA Navigation**: Uses `role="navigation"` and `aria-label="Breadcrumb"`
- **Semantic HTML**: Implements proper `<nav>` and `<ol>` structure
- **Current Page**: Uses `aria-current="page"` for the last item
- **Keyboard Navigation**: All interactive elements are keyboard accessible
- **Screen Reader**: Clear announcement of navigation structure
- **Focus Management**: Proper focus indicators and tab order

## Apple HIG Guidelines

### Visual Design

- **Hierarchy**: Clear visual hierarchy with proper typography
- **Depth**: Subtle glass effects that enhance without overwhelming
- **Color**: Consistent color usage for interactive and static elements
- **Spacing**: Appropriate spacing between items and separators

### Navigation

- **Clarity**: Clear indication of current location in hierarchy
- **Consistency**: Consistent behavior across the application
- **Efficiency**: Quick navigation to any level in the hierarchy
- **Context**: Provides context about user's location

### Interaction

- **Feedback**: Clear hover and focus states for interactive elements
- **Touch Targets**: Appropriate size for touch interaction
- **Animation**: Smooth transitions and micro-interactions
- **States**: Clear visual states for different interaction modes

### Content

- **Conciseness**: Keep labels short and descriptive
- **Consistency**: Use consistent naming conventions
- **Hierarchy**: Reflect the actual information architecture
- **Truncation**: Handle long paths gracefully

### Best Practices

- Use meaningful, descriptive labels
- Ensure all levels are clickable except the current page
- Consider mobile responsiveness for long breadcrumb trails
- Test with screen readers and keyboard navigation
- Maintain consistent styling with your navigation system
- Use appropriate separators that match your design language
- Consider truncation for very deep hierarchies
- Provide clear visual feedback for interactive elements
