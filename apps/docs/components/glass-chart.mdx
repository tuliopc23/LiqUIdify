## Overview

The Glass Chart components provide beautiful data visualization with Apple-inspired liquid glass effects. The collection includes LineChart, BarChart, and DonutChart components, each optimized for different data presentation needs.

## Components

### LineChart

Perfect for showing trends and changes over time.

```tsx
<LineChart
  data={data}
  width={400}
  height={200}
  strokeWidth={3}
  showDots
  gradient
  animated
/>
```

### BarChart

Ideal for comparing values across categories.

```tsx
<BarChart
  data={data}
  width={400}
  height={200}
  orientation="vertical"
  showValues
  animated
/>
```

### DonutChart

Great for showing proportions and percentages.

```tsx
<DonutChart
  data={data}
  width={200}
  height={200}
  innerRadius={60}
  showLabels
  centerContent={<CustomContent />}
/>
```

## Props

### Base Chart Props

| Prop          | Type               | Default | Description               |
| ------------- | ------------------ | ------- | ------------------------- |
| `data`        | `ChartDataPoint[]` | -       | **Required.** Chart data  |
| `width`       | `number`           | `400`   | Chart width in pixels     |
| `height`      | `number`           | `200`   | Chart height in pixels    |
| `className`   | `string`           | -       | Additional CSS classes    |
| `animated`    | `boolean`          | `true`  | Enable animations         |
| `showTooltip` | `boolean`          | `true`  | Show interactive tooltips |

### LineChart Props

| Prop          | Type      | Default | Description          |
| ------------- | --------- | ------- | -------------------- |
| `strokeWidth` | `number`  | `3`     | Line stroke width    |
| `showDots`    | `boolean` | `true`  | Show data point dots |
| `gradient`    | `boolean` | `true`  | Show gradient fill   |

### BarChart Props

| Prop          | Type                         | Default      | Description       |
| ------------- | ---------------------------- | ------------ | ----------------- |
| `orientation` | `'vertical' \| 'horizontal'` | `'vertical'` | Bar orientation   |
| `showValues`  | `boolean`                    | `true`       | Show value labels |

### DonutChart Props

| Prop            | Type        | Default | Description             |
| --------------- | ----------- | ------- | ----------------------- |
| `innerRadius`   | `number`    | `60`    | Inner radius in pixels  |
| `showLabels`    | `boolean`   | `true`  | Show segment labels     |
| `centerContent` | `ReactNode` | -       | Content for center area |

### ChartDataPoint Interface

```typescript
interface ChartDataPoint {
  label: string;
  value: number;
  color?: string;
  metadata?: Record<string, unknown>;
}
```

## Usage Patterns

### Real-time Data Updates

```tsx
const [data, setData] = useState(initialData)

useEffect(() => {
  const interval = setInterval(() => {
    setData(prevData => [
      ...prevData.slice(1),
      { label: new Date().toLocaleTimeString(), value: Math.random() * 100 }
    ])
  }, 1000)

  return () => clearInterval(interval)
}, [])

<LineChart data={data} animated />
```

### Custom Color Schemes

```tsx
const data = [
  { label: 'Success', value: 85, color: '#10B981' },
  { label: 'Warning', value: 12, color: '#F59E0B' },
  { label: 'Error', value: 3, color: '#EF4444' }
]

<DonutChart data={data} />
```

### Responsive Charts

```tsx
const [dimensions, setDimensions] = useState({ width: 400, height: 200 })

useEffect(() => {
  const handleResize = () => {
    const container = containerRef.current
    if (container) {
      setDimensions({
        width: container.offsetWidth,
        height: container.offsetHeight
      })
    }
  }

  window.addEventListener('resize', handleResize)
  return () => window.removeEventListener('resize', handleResize)
}, [])

<LineChart
  data={data}
  width={dimensions.width}
  height={dimensions.height}
/>
```

## Accessibility

### WCAG Compliance

- **Alternative Text**: Charts include descriptive titles and labels
- **Color Independence**: Information is not conveyed by color alone
- **Keyboard Navigation**: Interactive elements are keyboard accessible
- **Screen Readers**: Proper ARIA labels and data tables for complex charts

### Implementation Details

- Charts include `<title>` elements for screen readers
- Data is available in tabular format via ARIA
- Color contrast meets WCAG AA standards
- Interactive elements have proper focus indicators

### Data Tables for Screen Readers

```tsx
<div>
  <LineChart data={data} aria-describedby="chart-table" />

  <table id="chart-table" className="sr-only">
    <caption>Monthly Revenue Data</caption>
    <thead>
      <tr>
        <th>Month</th>
        <th>Revenue</th>
      </tr>

        <tr key={item.label}>
          <td>{item.label}</td>
          <td>{item.value}</td>
        </tr>
      ))}
    </tbody>
  </table>
</div>
```

## Apple HIG Guidelines

### Visual Design

- **Liquid Glass Effects**: Multi-layered transparency with backdrop blur
- **Color Harmony**: Uses Apple's color palette and semantic colors
- **Typography**: SF Pro font family for labels and values
- **Spacing**: Consistent 8pt grid system

### Data Visualization Principles

- **Clarity**: Data is presented clearly without unnecessary decoration
- **Accuracy**: Visual representation accurately reflects the data
- **Efficiency**: Users can quickly understand the information
- **Aesthetics**: Beautiful design that enhances comprehension

### Animation Guidelines

- **Purposeful Motion**: Animations help users understand data changes
- **Smooth Transitions**: Spring-based animations with proper easing
- **Performance**: 60fps animations that don't block interaction
- **Accessibility**: Respects user's motion preferences

### Color Usage

- **Semantic Colors**: Red for negative, green for positive, blue for neutral
- **Accessibility**: High contrast ratios for all color combinations
- **Consistency**: Consistent color usage across all chart types
- **Adaptation**: Colors adapt to light/dark mode preferences

## Best Practices

### Performance

- Use `React.memo` for charts with static data
- Implement data virtualization for large datasets
- Optimize SVG rendering for complex charts
- Use `requestAnimationFrame` for smooth animations

### Data Preparation

```tsx
// Memoize expensive data transformations
const chartData = useMemo(() => {
  return rawData.map((item) => ({
    label: formatLabel(item.date),
    value: calculateValue(item),
    color: getColorForValue(item.value),
  }));
}, [rawData]);
```

### Error Handling

```tsx
const ChartWithErrorBoundary = ({ data }) => {
  if (!data || data.length === 0) {
    return (
      <div className="flex items-center justify-center h-48">
        <p className="text-liquid-text-inverse/60">No data available</p>
      </div>
    );
  }

  return <LineChart data={data} />;
};
```

### Responsive Design

- Use percentage-based dimensions when possible
- Implement breakpoint-specific chart configurations
- Consider mobile-specific interaction patterns
- Optimize touch targets for mobile devices

### Loading States

```tsx
const ChartWithLoading = ({ data, loading }) => {
  if (loading) {
    return (
      <div className="animate-pulse">
        <div className="h-48 bg-liquid-bg/10 rounded-xl"></div>
      </div>
    );
  }

  return <LineChart data={data} />;
};
```
