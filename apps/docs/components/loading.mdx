# Loading

The `GlassLoading` component provides elegant loading indicators with glassmorphism effects. It supports multiple variants including spinners, dots, pulse, and bars with customizable sizes and text labels.

## Live Examples

### Basic Usage

```tsx
export default function Demo() {
  return (
    <div
      style={{
        display: "flex",
        gap: 24,
        flexWrap: "wrap",
        alignItems: "center",
      }}
    >
      <GlassLoading variant="spinner" size="md" />
      <GlassLoading variant="dots" size="md" />
      <GlassLoading variant="pulse" size="md" />
      <GlassLoading variant="bars" size="md" />
    </div>
  );
}
```

### With Text Labels

### Interactive Demo

```tsx
export default function Demo() {
  const [variant, setVariant] = useState("spinner");
  const [size, setSize] = useState("md");
  const [showText, setShowText] = useState(true);

  const variants = ["spinner", "dots", "pulse", "bars"];
  const sizes = ["sm", "md", "lg", "xl"];

  return (
    <div style={{ display: "flex", flexDirection: "column", gap: 24 }}>
      <div style={{ display: "flex", gap: 12, flexWrap: "wrap" }}>
        {variants.map((v) => (
          <GlassButton
            key={v}
            variant={variant === v ? "primary" : "ghost"}
            size="sm"
            onClick={() => setVariant(v)}
          >
            {v}
          </GlassButton>
        ))}
      </div>

      <div style={{ display: "flex", gap: 12, flexWrap: "wrap" }}>
        {sizes.map((s) => (
          <GlassButton
            key={s}
            variant={size === s ? "primary" : "ghost"}
            size="sm"
            onClick={() => setSize(s)}
          >
            {s.toUpperCase()}
          </GlassButton>
        ))}
      </div>

      <div>
        <GlassButton
          variant={showText ? "primary" : "ghost"}
          size="sm"
          onClick={() => setShowText(!showText)}
        >
          {showText ? "Hide Text" : "Show Text"}
        </GlassButton>
      </div>

      <div style={{ padding: 32, display: "flex", justifyContent: "center" }}>
        <GlassLoading
          variant={variant}
          size={size}
          text={showText ? "Loading content..." : undefined}
        />
      </div>
    </div>
  );
}
```

## Props

| Prop        | Type                                       | Default     | Description                             |
| ----------- | ------------------------------------------ | ----------- | --------------------------------------- |
| `variant`   | `"dots" \| "spinner" \| "pulse" \| "bars"` | `"spinner"` | Loading animation style                 |
| `size`      | `"sm" \| "md" \| "lg" \| "xl"`             | `"md"`      | Size of the loading indicator           |
| `text`      | `string`                                   | `undefined` | Optional text label below the indicator |
| `className` | `string`                                   | `undefined` | Additional CSS classes                  |

## Usage Patterns

### Page Loading

```tsx
// Full page loading state
<div className="fixed inset-0 flex items-center justify-center bg-black/20 backdrop-blur-sm">
  <GlassLoading variant="spinner" size="lg" text="Loading application..." />
</div>
```

### Button Loading State

```tsx
// Inline with buttons
<GlassButton disabled>
  <GlassLoading variant="spinner" size="sm" className="mr-2" />
  Processing...
</GlassButton>
```

### Content Loading

```tsx
// Content area loading
<div className="min-h-[200px] flex items-center justify-center">
  <GlassLoading variant="dots" text="Loading content..." />
</div>
```

### Data Processing

```tsx
// For data operations
<GlassLoading variant="bars" size="md" text="Analyzing data..." />
```

## Accessibility

### WCAG Compliance

- **WCAG 2.1 AA**: Fully compliant with color contrast and motion preferences
- **Screen Readers**: Properly announced with semantic markup
- **Motion Sensitivity**: Respects `prefers-reduced-motion` settings
- **Focus Management**: Non-interactive, doesn't interfere with focus flow

### Implementation Details

```tsx
// Accessible loading with announcements
<div role="status" aria-live="polite">
  <GlassLoading variant="spinner" text="Loading content..." />
  <span className="sr-only">Loading, please wait</span>
</div>
```

### Best Practices

- Always provide meaningful text labels for screen readers
- Use appropriate `aria-live` regions for dynamic loading states
- Consider motion preferences with `prefers-reduced-motion`
- Ensure sufficient color contrast for the loading indicators
- Provide alternative text descriptions for complex loading states

## Apple HIG Guidelines

### Visual Design

- **Clarity**: Clean, minimal loading indicators that don't overwhelm content
- **Deference**: Subtle animations that indicate progress without distraction
- **Depth**: Glassmorphism effects create visual hierarchy and focus

### Animation Principles

- **Smooth Transitions**: 60fps animations with hardware acceleration
- **Purposeful Motion**: Each variant serves a specific use case
- **Respectful**: Honors system motion preferences

### Usage Guidelines

- **Spinner**: General loading, network requests
- **Dots**: Sequential processing, multi-step operations
- **Pulse**: Heartbeat-like operations, real-time sync
- **Bars**: Data processing, analysis operations

### Size Recommendations

- **Small (sm)**: Inline elements, buttons, form fields
- **Medium (md)**: Default for most content areas
- **Large (lg)**: Primary loading states, full sections
- **Extra Large (xl)**: Full-page loading, splash screens

## Performance

### Optimization Features

- **CSS Animations**: Hardware-accelerated transforms
- **Minimal DOM**: Lightweight component structure
- **Memory Efficient**: No JavaScript animation loops
- **Responsive**: Adapts to different screen sizes

- Use CSS animations instead of JavaScript for better performance
- Implement proper cleanup for loading states
- Consider using skeleton screens for longer loading times
- Provide immediate feedback for user interactions
