## WCAG Compliance

LiqUIdify components are designed to meet WCAG 2.1 AA standards:

### Level A Requirements

- **Alt Text**: All decorative images use appropriate alt attributes
- **Keyboard Access**: All functionality available via keyboard
- **Focus Indicators**: Clear visual focus indicators on all interactive elements

### Level AA Requirements

- **Color Contrast**: Minimum 4.5:1 contrast ratio for normal text
- **Text Resize**: Text scalable up to 200% without horizontal scrolling
- **Focus Order**: Logical tab order throughout interfaces

## Built-in Accessibility Features

### Automatic ARIA

Components automatically include appropriate ARIA attributes:

```tsx
<GlassButton aria-pressed={isPressed}>
  Toggle Button
</GlassButton>

<GlassModal aria-labelledby="modal-title" aria-describedby="modal-description">
  <h2 id="modal-title">Modal Title</h2>
  <p id="modal-description">Modal description</p>
</GlassModal>
```

### Focus Management

Advanced focus management for complex components:

```tsx
import { GlassFocusTrap } from "@liquidify/components";

<GlassFocusTrap>
  <GlassModal>{/* Focus automatically trapped within modal */}</GlassModal>
</GlassFocusTrap>;
```

### Screen Reader Support

Components provide rich semantic information:

```tsx
<GlassProgress
  value={75}
  max={100}
  aria-label="Upload progress"
  aria-describedby="progress-description"
>
  <span id="progress-description">Uploading file: 75% complete</span>
</GlassProgress>
```

## Testing Accessibility

### Automated Testing

Use the built-in accessibility testing utilities:

```tsx
import { testAccessibility } from "@liquidify/testing";

test("component is accessible", async () => {
  const { container } = render(<MyComponent />);
  const results = await testAccessibility(container);
  expect(results).toHaveNoViolations();
});
```

### Manual Testing Checklist

<Steps>
  <Step title="Keyboard Navigation">
    Test all interactive elements using only the keyboard (Tab, Enter, Space,
    Arrow keys)
  </Step>
  <Step title="Screen Reader">
    Test with a screen reader (VoiceOver, NVDA, or JAWS) to ensure proper
    announcements
  </Step>
  <Step title="High Contrast">
    Enable high contrast mode to verify component visibility
  </Step>
  <Step title="Zoom Testing">
    Test at 200% zoom to ensure content remains accessible
  </Step>
</Steps>

## Component-Specific Guidelines

### Forms

- Use proper labels and descriptions
- Provide clear error messages
- Group related inputs with fieldsets

```tsx
<GlassFormField>
  <GlassLabel htmlFor="email">Email Address</GlassLabel>
  <GlassInput
    id="email"
    type="email"
    aria-describedby="email-error"
    aria-invalid={hasError}
  />
  {hasError && (
    <GlassErrorMessage id="email-error">
      Please enter a valid email address
    </GlassErrorMessage>
  )}
</GlassFormField>
```

### Navigation

- Provide skip links for main content
- Use semantic navigation landmarks
- Implement proper heading hierarchy

### Interactive Elements

- Ensure minimum touch target size (44px)
- Provide loading states and feedback
- Use appropriate ARIA live regions for dynamic content
